# Clean Formal Run Plan

更新时间：2026-04-01

## 目标

本轮 clean run 只服务于正式论文证据更新，目标是将当前 `paper_exec` 中的 blocked 结果替换为 protocol-clean 结果。

## 固定协议

- 训练阶段开启：`--disable_train_test_eval`
- checkpoint 选择：仅基于验证集
- 测试集：仅在训练结束、checkpoint 冻结后使用
- retrieval 超参数：固定为 `k=5, beta=0.5, late_q=0.80`
- retrieval library：仅由 train fold 构建
- val/test stride：固定为 `1`
- 禁止使用：stage-align exploratory 分支、gated retrieval、test-fold tuning

## 最小正式实验包

### A. TimerXL backbone only

- fold1: `c1,c4 -> c6`
- fold2: `c4,c6 -> c1`
- fold3: `c1,c6 -> c4`
- 结果目录：`results/20260401_BaselineClean_seqgpu0`

### B. TimerXL + TMA

- fold1: `c1,c4 -> c6`
- fold2: `c4,c6 -> c1`
- fold3: `c1,c6 -> c4`
- 结果目录：`results/20260401_TMAClean_seqgpu2`

### C. TimerXL + TMA + retrieval backbone

- fold1: `c1,c4 -> c6`
- fold2: `c4,c6 -> c1`
- fold3: `c1,c6 -> c4`
- 结果目录：`results/20260401_RetrievalBackboneClean_seqgpu3`
- 说明：该阶段先训练 retrieval 使用的 backbone checkpoint，随后再执行 clean retrieval inference

## GPU 分配

考虑到当前服务器显存占用情况，本轮采用“每个方法分支占一张 GPU、三折顺序跑”的策略：

- baseline: GPU 0
- TMA: GPU 2
- retrieval backbone: GPU 3

GPU 1 当前外部任务占用过高，暂不纳入 clean run。

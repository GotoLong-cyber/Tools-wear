# 协议补丁说明

更新时间：2026-04-01

## 目标

将正式的 PHM2010 训练流程从 `训练 + 验证 + 每个 epoch 监控测试集`
改为仅包含以下内容的 clean protocol：

- 训练集用于优化
- 验证集用于 checkpoint 选择
- 测试集仅在 checkpoint 冻结后使用

## 修改文件

### 1. `run.py`

新增专用 formal-protocol 开关：

- `--disable_train_test_eval`

语义如下：

- 开启后，`Exp_Forecast.train()` 在训练期间不得访问 `test_loader`
- 训练结束后的 `exp.test(setting)` 保留不变

### 2. `exp/exp_forecast.py`

对 `Exp_Forecast.train()` 做了补丁：

- 补丁前：
  - 训练时始终构建 `test_data, test_loader`
  - 每个 epoch 都计算 `test_loss`
  - checkpoint 选择阶段同时打印 `Vali Loss` 与 `Test Loss`
- 补丁后：
  - 是否进行 epoch 级测试评估由 `not args.disable_train_test_eval` 控制
  - 在 formal clean run 中，训练期间不会构建 `test_loader`
  - checkpoint 选择只依赖验证集损失
  - 测试评估仍在后续 `run.py -> exp.test(setting)` 中执行

## 为什么这个补丁是协议安全的

- 它不改变优化目标
- 它只是在 formal 路径中移除了一个易引发质疑的监控通道
- 对未传入该开关的 exploratory run，旧行为仍被保留
- 它显式区分了：
  - legacy / blocked checkpoints
  - clean formal checkpoints

## Formal 使用规则

所有 clean paper run 都必须传入：

- `--disable_train_test_eval`

未使用该开关训练出来的 legacy checkpoint 仍然属于：

- 最终论文证据中的 protocol-blocked 结果
- 只能作为 deprecated / exploratory comparison 使用

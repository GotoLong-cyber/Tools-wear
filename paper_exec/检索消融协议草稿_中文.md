# 检索消融与参数协议草稿（中文）

## 5.4 检索消融与超参数协议

表 5 给出了正式检索模式消融结果。整体上看，单纯移除检索时，retrieval backbone 的 head-only 三折 MAE 分别为 `9.24558162689209` μm、`4.199134349822998` μm 和 `4.8541717529296875` μm；在引入基于未来增量的残差修正后，正式主方案 `delta-blend@k10_b07` 对应的三折 MAE 下降到 `3.2064666748046875` μm、`3.3330025672912598` μm 和 `2.995772123336792` μm。该结果说明，检索模块的有效性并不来自简单地把历史邻居的绝对磨损值直接搬运到测试样本上，而是来自对基础预测所隐含的未来磨损增量进行残差修正。

不同检索模式之间的比较也支持这一结论。`knn-only@k10` 在 fold1 上的 MAE 高达 `25.83344268798828` μm，明显不适合作为正式主线；而 `delta-knn-only@k10` 虽在部分折上取得更低的单折误差，例如在 fold1 和 fold3 上分别达到 `2.515413284301758` μm 和 `2.4314849376678467` μm，但它完全去除了基础预测头的约束，难以作为整体最平衡的主文方案。相比之下，`delta-blend@k10_b07` 在三折上都维持了稳定且较低的误差水平，并且与本文“TimerXL 提供基础趋势，KNN-DRR 修正未来增量残差”的方法叙事保持一致。因此，正式主结果采用 `delta-blend` 作为唯一主线模式，而不是把纯检索变体提升为论文的中心方案。

正式检索配置的确定过程也需要明确交代。本文并未在 outer test folds 上搜索 `k`、`beta` 或 `late_q`，而是采用 train-only 的 inner LOSO 协议完成超参数选择。对每个外层折，两个源工况在训练内部轮流扮演检索库工况与验证目标工况，在保留 cross-condition gap 与 wear-rate mismatch 的前提下比较候选参数组合。所有候选配置只在这些内层任务上聚合误差后进行排序，外层测试工况在参数确定前始终被封存。最终，正式主线固定采用 `k=10`、`beta=0.7`、`late_q=0.0`，并在三折 outer test 上各自仅评估一次。因而，表 5 中的所有正式检索结果都应被理解为“基于 inner LOSO train-only 选参后的一次性外层测试结果”，而不是“在外层测试上调优后得到的最优值”。

从 reviewer-facing 的角度看，这一写法具有两层含义。首先，本文的检索超参数来源是可追溯且可辩护的，因为其选择过程严格局限于源工况内部，不包含任何 outer-test tuning。其次，本文对 `delta-blend` 的采用并非出于事后挑选，而是出于正式协议下的整体平衡性考虑：它在稳定保留骨干网络全局趋势的同时，能够有效修正 late-stage future-increment residual bias，并与固定的方法层级保持一致。基于这一协议，本文的正式主线能够同时满足结果可信性与方法叙事完整性。

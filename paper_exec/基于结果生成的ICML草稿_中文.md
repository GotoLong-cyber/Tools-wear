# 基于训练时多步幅增强与 KNN 增量残差检索的小样本跨工况刀具磨损预测

## 摘要
小样本跨工况刀具磨损预测具有较高难度，其根源在于不同切削工况下的退化速率存在差异，导致固定时间步表示在跨域时常出现错位。在这一设定下，即便采用较强的时序骨干模型，仍可能出现结构性的 late-stage 未来磨损增量低估。本文在 PHM2010 上构建了三层管线：以 TimerXL 作为骨干预测器，以训练时多步幅增强（TMA）作为辅助表示增强机制，以基于 KNN 的增量残差检索（KNN-DRR）作为主要纠偏模块。TMA 仅在训练阶段生效，不使用任何测试工况信息；KNN-DRR 在推理阶段基于 train-only 检索库执行残差修正。超参数通过源工况内部的 inner leave-one-source-out 验证固定，外层测试折仅评估一次。在三折 leave-one-condition-out 评估下，平均 MAE 由 8.7711（TimerXL head-only）降至 6.1484（TimerXL+TMA），并进一步降至 3.1784（TimerXL+TMA+KNN-DRR）；平均 RMSE 由 11.9747 降至 9.4630，并进一步降至 3.9763。在代表性最难折（c1,c4 -> c6）上，fold 级 MAE 从 13.3984 降至 9.2761，再降至 3.2065。该折 late-stage MAE 通过 delta-blend 修正从 21.2180 降至 5.7781。上述结果支持清晰的分工：TimerXL 提供骨干趋势，TMA 提升跨工况可比性，KNN-DRR 执行主要残差修正。

## 1. 引言
真实制造场景中的刀具磨损预测通常受限于各工况下可获得的标注轨迹数量。在跨工况部署中，困难不仅来自特征分布差异，也来自退化节奏差异：相同时间索引在不同工况下并不对应同一磨损阶段。这种时间尺度错位在磨损后期危害尤为明显，因为未来增量加速增长，而常规预测器容易低估。

本文的出发点是：基础模型风格的时序骨干在该场景中有价值，但仍不足以单独解决问题。因此，我们将方案分为三层，而非将全部组件视为同等创新。第一层，TimerXL 提供基础时序表示与趋势预测。第二层，TMA 通过训练时多步幅增强提升速率差异下的表示可比性。第三层，KNN-DRR 使用 train-only 检索库对未来磨损增量执行显式残差修正。

该设计直接针对结构性失效模式，同时保持严格 clean protocol：不在 outer test 上调参，不发生测试标签泄露，并通过 inner train-only 验证确定全局固定检索配置。

### 贡献
1. 在 PHM2010 上，我们建立了小样本跨工况刀具磨损预测的正式证据链，并明确模块层级：TimerXL 为 backbone，TMA 为辅助增强，KNN-DRR 为主要创新。
2. 我们证明训练时时间尺度增强与检索式残差修正具有互补增益，在 clean protocol 约束下将平均 MAE 从 8.7711 降至 3.1784。
3. 我们提供分阶段证据，表明主导结构性误差是 late-stage 未来磨损增量低估，且残差修正可显著缓解该偏差。

## 2. 方法
### 2.1 问题设定
给定源工况历史窗口，模型预测未见目标工况下的未来磨损值。关键难点在于跨工况退化速率不匹配：相同时间索引可能对应不同物理磨损阶段。

### 2.2 TimerXL 骨干
TimerXL 用作骨干预测器，用于建模时序动力学并给出基础未来磨损趋势。在本文框架中，TimerXL 是平台模型而非主要创新点。

### 2.3 训练时多步幅增强（TMA）
TMA 是训练期辅助机制。它在训练窗口中扰动时间步幅，以在退化节奏存在差异时提升表示可比性。TMA 不是推理期对齐模块，也不会在验证或测试阶段执行自适应对齐。

### 2.4 KNN 增量残差检索（KNN-DRR）
KNN-DRR 是主要纠偏模块。在推理阶段，它从 train-only 检索库中检索相似历史轨迹，并在骨干趋势之上修正未来磨损增量残差。这是残差修正设计，而非替代骨干网络。

### 2.5 Train-Only 超参数协议
检索超参数通过源工况内部的 inner leave-one-source-out 验证固定。对每个 outer fold，源工况轮换作为建库域与验证域，并在 inner splits 上平均误差。随后将全局固定配置用于各 outer test fold 的一次性评估。最终正式配置为：k=10，beta=0.7，late_q=0.0。

## 3. 实验设置
### 3.1 数据集与跨工况协议
我们在 PHM2010 上评估，包含三个工况（c1, c4, c6），采用三折 leave-one-condition-out（LOCO）：
- fold1: c1,c4 -> c6
- fold2: c4,c6 -> c1
- fold3: c1,c6 -> c4

### 3.2 评价指标
主指标为全曲线 MAE 与 RMSE（um）。机制分析中，我们进一步考察代表性最难折表现与分阶段残差统计（early/mid/late）。

### 3.3 对比变体
我们报告正式三阶段链条：
- TimerXL head-only
- TimerXL + TMA
- TimerXL + TMA + KNN-DRR

全部主结果均遵循 clean protocol 与 train-only inner LOSO 选参。

## 4. 结果
### 4.1 三折主结果
三折 outer fold 平均性能：
- TimerXL head-only：MAE 8.7711，RMSE 11.9747
- TimerXL + TMA：MAE 6.1484，RMSE 9.4630
- TimerXL + TMA + KNN-DRR：MAE 3.1784，RMSE 3.9763

该链条表明存在两类互补增益：TMA 提升跨工况表示可用性，KNN-DRR 提供主要残差修正收益。

### 4.2 代表性最难折
在 fold1（c1,c4 -> c6）上，fold 级 MAE 变化为：
- 13.3984（head-only）
- 降至 9.2761（TimerXL+TMA）
- 再降至 3.2065（TimerXL+TMA+KNN-DRR）

该折最能暴露后期错配问题，并凸显显式残差修正的价值。

### 4.3 分阶段偏差证据
在 fold1 的 late stage，MAE 从 21.2180（head-only）降至 5.7781（delta-blend 修正）。平均残差从强负偏差（-20.4768）向显著减弱的低估（-5.7781）移动，支持“主导结构性误差是 late-stage 增量低估”的判断。

## 5. 讨论
我们的证据支持分工明确的解释：
- TimerXL 捕获骨干趋势信息。
- TMA 在时间尺度变化下提升表示可比性。
- KNN-DRR 执行未来增量残差偏差的主要修正。

这种分层解释对科学表达至关重要。观测到的收益不应被解释为对 backbone 的替代；更准确地说，在已有合理趋势预测器的前提下，检索式残差修正最有效。

此外，train-only inner LOSO 协议同样关键。它在不使用 outer test 调参的前提下保留了跨工况验证压力，提升了结果可信度与稳健性。

## 6. 局限性
本研究存在明确边界。

第一，当前正式验证仅在 PHM2010 上进行，因此结论应限定于该数据集与协议，而非泛化到所有工业预测任务。

第二，当前设定属于 history-aware wear forecasting。若扩展到严格 sensor-only blind prediction，还需要额外的磨损状态估计阶段。

第三，检索修正仍依赖 train-only 检索库中相关历史模式的覆盖；当 late-stage 覆盖极稀疏时，修正效果可能减弱。

## 7. 结论
本文提出了面向小样本跨工况刀具磨损预测的 clean-protocol 管线，并明确模块层级：TimerXL 为骨干，TMA 为训练期辅助增强，KNN-DRR 为主要残差修正模块。在 PHM2010 三折 LOCO 下，平均 MAE 从 8.7711 提升至 3.1784，最难折 late-stage MAE 从 21.2180 降至 5.7781。结果表明，关键瓶颈在于时间尺度错位下的结构性后期低估，而训练期可比性增强结合检索式残差修正是一种有效且协议一致的解决路径。

# 新协议实施计划

## 1. 当前统一结论

经过本轮重新清理实验协议、补跑特征线、重做 `epoch` 与 `KNN` 敏感性分析后，当前主线已经明确收束为：

- 主特征：`pure RMS7`
- 主协议：`wear_agg = mean`
- 训练方式：`no validation set`
- 当前最优训练轮数：`epoch = 300`
- 外层评测：三折 `LOCO`
- 当前最强模块链：`Baseline + KNN`

这意味着：

- 旧的 `RMS + 极差 + 频谱熵` 组合不再作为主线
- `TMA` 不再是终版主线中的必要模块
- 后续所有正文实验、图表和消融，都应围绕 `pure RMS7 + Baseline + KNN` 展开

---

## 2. 为什么主线回到 pure RMS7

### 2.1 旧轻量特征方案的问题

旧的轻量特征方案虽然早期结果较好，但存在明显论证缺口：

- 缺乏稳定、清晰的文献依据
- 通道选择解释不充分
- “为什么选这几个、为什么不选别的”难以自圆其说

因此，它不适合作为论文终版主线。

### 2.2 当前特征筛选后的结论

本轮实验已经在统一协议下比较了以下几类特征：

- `pure RMS7`
- `RMS7 + PTP7`
- `RMS7 + WAV7`
- `RMS7 + PTP7 + WAV7`
- `Full-133D`

当前结论是：

- `pure RMS7` 是最稳、最轻、最可解释的主线输入
- 加入 `PTP` 或 `WAV` 后，结果并未稳定优于 `pure RMS7`
- `Full-133D` 可作为行业级全量统计特征基线，但不是最终最佳输入方案

因此，终版论文的特征主线应明确为：

> `pure RMS7`

---

## 3. 当前模块结论

### 3.1 TMA 的定位

本轮实验表明：

- `TMA` 在某些分支上会带来局部改善
- 但在最终主线 `pure RMS7` 上，其独立贡献有限
- `Baseline + KNN` 已经优于 `TMA + KNN`

因此，`TMA` 不应再作为论文的第一主创新点，而应降级为：

- 辅助机制探讨
或
- Discussion 中的局限性分析

### 3.2 KNN 的定位

本轮实验已经明确：

- 真正稳定且主要的增益来自 `KNN`
- `knn-only` 在当前最终协议下优于 `blend`
- 所以当前最强主线应定义为：

> `TimerXL + KNN`

---

## 4. 当前最关键的合规缺口

虽然现在最终主线已经改成了 `Baseline + KNN`，但有一个论文级风险尚未完全消除：

> 当前使用的 `k / beta / late_q`，最初主要来自 `TMA / retrieval backbone` 路径上的 inner-LOSO 选参，而不是严格在最终主线 `Baseline + KNN` 上完成的。

这会引出一个审稿风险：

- 如果最终主线是 `Baseline + KNN`
- 那么超参数为什么不是在 `Baseline + KNN` 上选的？
- 是否存在路径不一致，甚至利用 outer test 反向调参的问题？

因此，后续最优先的任务不是扩展更多模型，而是：

> 在 `Baseline + KNN` 这条最终主线上，重新做一轮完全合法的 source-only inner-LOSO 超参数搜索。

---

## 5. 下一步实验主任务

### 第一任务：Baseline + KNN 主线合法选参

协议锁定：

- 特征：`pure RMS7`
- 训练协议：`no validation set`
- 训练轮数：`epoch = 300`
- 主体架构：`Baseline + KNN`
- 外层测试禁止参与选参

需扫描的参数：

- `k ∈ {3, 5, 7, 10}`
- `beta ∈ {0.3, 0.5, 0.7, 1.0}`
- `late_q ∈ {0.0, 0.5, 0.8}`

输出物：

- 一张新的 `Baseline + KNN` 主线选参图
- 一份新的最优参数配置
- 一段可直接写进论文的合法选参说明

### 第二任务：特征总表大扫除

当前总表仍包含：

- 不同 `epoch` 混排
- 部分旧探索结果
- `NaN`
- 不完全同频的特征对比

因此需要再清一遍，只保留论文终版真正需要的结果：

- `pure RMS7`
- `RMS7 + PTP7`
- `RMS7 + PTP7 + WAV7`
- `Full-133D`

且必须满足：

- 同一训练轮数
- 同一协议
- 无 `NaN`

---

## 6. 论文叙事的更新方向

当前论文主卖点不应再写成：

- `TMA + KNN`

而应转成：

### 主卖点 1：轻量物理表征

利用具有明确物理意义的 `RMS` 构造紧凑输入表示，避免高维全量统计特征带来的冗余与不稳定性。

### 主卖点 2：隐空间几何检索

在极少样本跨工况条件下，参数化回归头容易失稳，而非参数化 `KNN` 通过利用 `TimerXL` 隐空间的几何邻近性，实现更稳健的检索式修正。

### TMA 的角色调整

`TMA` 仍可保留，但只作为：

- 辅助机制
- 或 Discussion 中的局限性讨论

---

## 7. 当前最稳的对外口径

在正式完成 `Baseline + KNN` 主线合法选参之前，当前汇报应使用如下说法：

1. 主特征已经确定为 `pure RMS7`
2. 当前结果显示 `TimerXL + KNN` 是最强主线
3. `TMA` 不是必要模块
4. 但为了完全满足导师要求与审稿规范，下一步仍需在 `Baseline + KNN` 路径上重新完成一次 source-only inner-LOSO 选参

这句话最关键：

> 当前结果已经足够说明 KNN 是主要增益来源，但论文终版必须在最终主线上重新完成一次合法选参，才能完全闭环。

---

## 8. 现在的执行顺序

后续执行顺序固定为：

1. 先做 `Baseline + KNN` 主线的合法选参
2. 再做特征总表清理与终表重建
3. 最后统一改写论文实验章节、主张边界与摘要

不再建议做的事情：

- 继续把 `TMA` 路径选参图当成终版图
- 在总表中混排 `epoch=200` 与 `epoch=300`
- 把存在 `NaN` 的探索性结果直接放进论文终表

---

## 9. 一句话版总目标

> 用统一协议下的 `pure RMS7 + TimerXL + KNN`，替代旧的 `TMA` 主线，并通过合法选参、同频特征消融与叙事重构，把当前实验真正收成一篇可辩护的论文终版。

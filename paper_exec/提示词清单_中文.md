# 写作 Prompt 清单（最小可用）

> 适用范围：当前 `paper_exec` 主线写作与收口。  
> 使用原则：不新增实验、不改正式数字、不越过主张边界。

## 1. 作者口吻重写（中文正文段）

```text
你是中文学术写作编辑。请把我给你的段落重写成“作者本人语气”，要求：
1) 保留全部事实与数字，不新增实验，不改结论方向；
2) 句子更自然，避免模板化和AI腔；
3) 输出只给“重写后正文”，不要解释。
这是我的段落：
【粘贴段落】
```

## 2. 主张边界审计（防越界）

```text
你是论文主张审计助手。请按“允许/风险/禁止”三栏审计下面文本。
硬约束：
- TimerXL=backbone；TMA=auxiliary enhancement；KNN-DRR=main innovation
- 不把inner LOSO写成outer test调参
- 不把代表性困难场景写成设计先验
- 不夸大为“普适所有任务”
输出格式：
A. 允许保留句子
B. 风险句子+改写建议
C. 禁止句子+删除理由
待审文本：
【粘贴段落】
```

## 3. 数字一致性核对（结果段/摘要/结论）

```text
请检查下文数字是否与“目标数字表”一致，只做核对不改写。
目标数字表：
- 三折平均MAE：8.7711 / 6.1484 / 3.1784
- 三折平均RMSE：11.9747 / 9.4630 / 3.9763
- 代表性困难场景late-stage MAE：21.2180 -> 5.7781
输出：
1) 一致项
2) 不一致项（逐条指出原句）
3) 最小修正建议（只给替换后的数字）
待核对文本：
【粘贴文本】
```

## 4. 讨论段“去夸大+保贡献”

```text
请把这段 Discussion 改成“审稿人可接受”的稳健写法：
要求：
1) 保留贡献，不降级为无效工作；
2) 明确边界：单数据集、history-aware、检索库覆盖依赖；
3) 不新增实验承诺，不引入新数据结果；
4) 输出一个完整段落（中文，120-220字）。
原文：
【粘贴段落】
```

## 5. 摘要压缩到可投稿版本（中文）

```text
请将以下中文摘要压缩为 180-230 字投稿版。
要求：
1) 保留问题-方法-协议-核心结果四要素；
2) 数字必须原样保留；
3) 不使用夸张词（如“显著领先所有方法”）；
4) 输出仅摘要正文。
原文：
【粘贴摘要】
```

## 6. Reviewer 视角快速预审（整段）

```text
你是严格审稿人。请只找“会被质疑的点”，不要泛泛表扬。
输出格式固定：
- Major Concern（最多3条）
- Minor Concern（最多5条）
- 可立即修复清单（按优先级1-5）
约束：不得建议新增实验；只允许文本层修复。
待评文本：
【粘贴方法/结果/讨论段】
```

## 快速使用顺序（建议）

1. 先用“3. 数字一致性核对”锁数字。
2. 再用“2. 主张边界审计”锁边界。
3. 然后用“1. 作者口吻重写”做你自己的文风版本。
4. 最后用“6. Reviewer 视角快速预审”做提交前检查。

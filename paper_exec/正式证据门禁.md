# 正式证据闸门

更新时间：2026-04-01

本文件回答一个问题：  
“一个资产什么时候可以被视为正式证据并进入论文正文？”

## 1. 主结果 CSV 的闸门条件

适用对象：

- `main_results_3fold.csv`
- `hardest_scenario_results.csv`
- `retrieval_ablation.csv`
- `stage_residual_compare.csv`
- `retrieval_quality_before_after_a2.csv`
- `tma_ablation.csv`

必须同时满足：

1. 数据来自 clean checkpoint
2. retrieval 参数来自 inner LOSO train-only 选择
3. outer test 只评估一次
4. 未混入 `cleanfixed` / `trainonlyselect` / 更早 blocked 结果
5. 与 `paper_exec清单.md`、`论文资产看板.md`、`执行日志.md` 一致

只要有一项不满足：

- 一律不得进入正文主表

## 2. 主图的闸门条件

适用对象：

- Fig.1–Fig.7

必须同时满足：

1. 图中使用的数据对应当前正式 CSV
2. 图注不夸大主张
3. 图中 hardest scenario 仅作为实验发现，不作为设计先验
4. 图中不引用已弃用结果目录

只要有一项不满足：

- 一律不得进入正文主图

## 3. 结果描述文本的闸门条件

适用对象：

- `结果片段_中文.md`
- `结果片段_英文.md`
- `最终结果段落_中文.md`
- `最终结果段落_英文.md`

必须同时满足：

1. 数字与正式 CSV 完全一致
2. 叙事符合固定层级：
   - TimerXL = backbone
   - TMA = auxiliary enhancement
   - KNN-DRR = main innovation
3. 不把 inner LOSO 写成 outer test 选参
4. 不把 KNN-DRR 写成替代 backbone

只要有一项不满足：

- 一律不得复制进正文

## 4. 可选增强资产的闸门条件

适用对象：

- `backbone_comparison`
- `full133_vs_selected`
- `Table A2 / Fig.A1`

判定规则：

- 若结果不完整，但不影响主线，可以保留为附录或暂不使用
- 不完整的可选资产不得抬升为主结论证据

## 5. 当前立即可执行的闸门动作

1. 逐项核对主线 CSV 与 `20260401_RetrievalV21_innerlososelect`
2. 逐项核对 Fig.1–Fig.7 的数据来源与 caption
3. 逐项核对 `final_result_paragraphs_*` 与当前正式数字
4. 把不满足闸门条件的资产显式标注为“参考 / 附录 / 不纳入正文”

## 6. 闸门通过后的写作权限

只有在闸门通过后，以下内容才允许直接进入正文：

- 方法节中的参数选择协议描述
- 实验节中的主结果、困难场景、retrieval ablation、TMA 机制解释
- 摘要与结论中的关键数字

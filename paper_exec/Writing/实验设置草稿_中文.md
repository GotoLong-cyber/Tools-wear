# 实验设置草稿（中文）

## 4. 实验设置

本文在 PHM2010 刀具磨损预测数据集上评估所提方法。该数据集包含三个典型工况 `c1`、`c4` 和 `c6`，每个工况均对应一条完整的刀具磨损演化轨迹，单条轨迹长度均为 315 个时间步，磨损标签以微米为单位给出。由于不同工况下切削条件不同，同一切削步对应的实际磨损阶段并不一致，因此 PHM2010 是检验跨工况时间尺度错位与晚期磨损低估问题的合适基准。按照本文锁定的正式协议，我们采用三折 leave-one-condition-out（LOCO）评估：fold1 为 `c1,c4 -> c6`，fold2 为 `c4,c6 -> c1`，fold3 为 `c1,c6 -> c4`。在每个外层折中，两个源工况用于训练与验证，剩余一个工况被完全封存并作为最终测试对象。

训练与评估边界遵循严格的 clean protocol。骨干网络训练阶段只访问源工况样本，并在源工况内部执行训练/验证划分；训练过程中的模型选择基于验证误差完成，不进行任何 epoch 级测试监控。TMA 仅在训练阶段作用于历史窗口增强，对应的训练步长为 `stride ∈ {1,2}`，其中额外步长以随机方式作用于训练批次；验证与测试始终保持 `stride=1`。因此，测试工况在整个训练和模型选择过程中始终被封存，只有当骨干网络参数完全冻结后才会被用于离线评估。所有正式结果均采用这一协议，正式指标不混入历史 blocked 结果或 exploratory 分支结果。

本文的评价指标为全曲线平均绝对误差（fullcurve raw MAE）与全曲线均方根误差（fullcurve raw RMSE），单位均为微米。对于机制分析，本文还报告代表性困难场景和分阶段误差分解，包括 early、mid、late 三个阶段的 MAE、RMSE、mean residual 以及低估比例。这些附加指标并不改变主结果的判断标准，而是用于解释正式主结果为何成立，尤其用于验证“late-stage future-increment underestimation 是主要结构性误差来源”这一机制性主张。

在 retrieval 参数选择方面，本文采用 train-only 的 inner leave-one-source-out（inner LOSO）协议，而不是在 outer test fold 上进行调参。具体来说，对于每个外层折，设两个源工况分别为 A 和 B，则先使用 A 构建检索库并以 B 作为内层验证目标，再交换 A 与 B 的角色重复一次。所有候选的 `k`、`beta` 与 `late_q` 组合均只在这些内层 cross-condition 验证任务上比较，外层测试工况不参与任何形式的参数选择。完成内层验证后，我们聚合全部 inner splits 的误差并确定全局固定配置，然后在三折 outer test 上各自仅评估一次。按照当前正式主线的锁定结果，最终外层测试统一使用 `k=10`、`beta=0.7`、`late_q=0.0`。

在这一协议下，外层测试与内层选参的关系需要明确区分。inner LOSO 的任务是模拟“源工况之间的跨工况泛化”，以便在不触碰 outer test 的前提下选择更接近真实应用场景的检索超参数；outer test 的任务则是对冻结后的完整方法进行最终一次性评估。换言之，inner LOSO 只决定参数，不提供正式主结果；正式主结果完全来自 outer LOCO 测试。基于这一 clean protocol，本文最终形成三条可直接比较的正式结果链：TimerXL head-only、TimerXL + TMA，以及 TimerXL + TMA + KNN-DRR。其中，KNN-DRR 在 inner LOSO 选出的固定配置下取得三折平均 MAE `3.1784` μm 和平均 RMSE `3.9763` μm，作为本文主结果的唯一正式来源。

# 方法与实验主张检查表

本文档用于核对 `方法章节草稿_中文.md` 与 `实验设置草稿_中文.md` 是否违反已锁定的主张边界。

## 1. 是否把 TimerXL 夸大为主要创新

检查结果：未违反。  
说明：方法草稿中将 TimerXL 明确表述为“骨干网络”“基础时序预测平台”，并明确指出其并不直接解决跨工况时间尺度错位或晚期未来增量低估，因此未把 TimerXL 上升为主要创新。

## 2. 是否把 TMA 描述成最终预测来源

检查结果：未违反。  
说明：方法草稿把 TMA 定义为训练期时间尺度增强，强调其主要作用是提升跨速率条件下的表示可比性和检索可用性；实验设置中也明确指出验证与测试保持 `stride=1`，因此不存在把 TMA 写成最终预测源的问题。

## 3. 是否把 KNN-DRR 描述成替代骨干网络

检查结果：未违反。  
说明：方法草稿明确说明 KNN-DRR 在 TimerXL 基础预测之上执行 future wear increment residual correction，而不是用检索直接替代基础预测器。文中使用的是“残差修正”“定向纠偏”等表述，没有出现“replace the backbone”式写法。

## 4. 是否把 inner LOSO 写成 outer-test tuning

检查结果：未违反。  
说明：实验设置草稿中明确区分了 inner LOSO 与 outer LOCO test 的角色：inner LOSO 仅用于源工况内部 train-only 超参数选择，outer test 在全局固定配置确定后只评估一次。文中没有把 `k=10, beta=0.7, late_q=0.0` 描述为 outer test 最优选取结果。

## 5. 是否把代表性困难场景写成设计先验

检查结果：未违反。  
说明：方法草稿未以 hardest fold 作为方法设计出发点；实验设置草稿也只将困难场景相关指标作为结果解释与机制分析工具，而未将其写成先验目标或设计驱动。

## 6. 是否混入 exploratory evidence

检查结果：未违反。  
说明：两份草稿均以 `paper_exec/final_main_assets/` 中锁定的主文图表、正式 CSV、正式结果段落和主张边界文件为依据，没有引用 stage-align exploratory 分支、blocked 结果目录、旧的 cleanfixed 结果或 appendix-only 的不完整增强证据作为正式主结果。

## 7. 总体判断

当前两份草稿与固定主线保持一致：

- TimerXL = backbone/platform
- TMA = auxiliary train-time enhancement
- KNN-DRR = main innovation / retrieval residual correction
- inner LOSO = train-only hyperparameter selection protocol
- hardest scenario = 实验发现而非设计先验
- 正式证据 = clean protocol + inner LOSO + outer test 单次评估

因此，就当前版本而言，方法节与实验设置节均未越过既定的主张边界，可继续作为正文写作底稿使用。

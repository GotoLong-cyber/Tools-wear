# 论文级闭环收口计划_最终版

## 0. 当前阶段的统一判断

截至目前，实验主线已经发生了明确转移：

- 主特征：`pure RMS7`
- 主协议：`wear_agg = mean`，`no validation set`，`LOCO`
- 当前最强模块链：`Baseline + KNN`
- `TMA` 不再是最终主线中的必要模块

因此，后续所有“论文终版”实验与图表，都必须围绕以下最终主线展开：

> `pure RMS7 + Baseline + KNN`

如果后续仍沿用 `TMA` 路径上的超参数选择结果，审稿人会直接质疑：

- 这些超参数是不是在另一条模型路径上选出来的？
- 如果最终主线已经改成 `Baseline + KNN`，那 `k / beta / late_q` 为什么不是在这条主线上选的？
- 是否存在协议不一致，或者利用 outer test 结果倒推参数的问题？

所以现在必须做的，不是继续扩展实验，而是完成“最终协议对齐”与“主卖点重构”。

## 2. 第二阶段：特征消融总表同频大扫除

### 2.1 目标

清理当前总表中的历史包袱，生成一张**没有 NaN、没有旧协议、没有混杂设置**的论文级特征消融总表。

### 2.2 当前痛点

目前已有总表虽然对实验追踪有帮助，但还不适合作为论文终版特征消融表，原因包括：

- `full133` 还没有完全对齐到最终协议
- `21维` 组合特征存在 `NaN`
- `epoch=200` 与 `epoch=300` 混在同一总表中
- 某些结果来自探索期设置，某些来自最终协议，严格上不可横向并列

导师明确反对这种“关公战秦琼”式比较。

### 2.3 清理原则

凡是不满足最终主协议的旧数据，都应：

- 删除
或
- 补齐后再进入论文表

最终论文特征消融表，只保留**同一协议、同一轮数、同一主线**下的结果。

### 2.4 统一协议

用于论文级特征消融的协议统一为：

- `pure RMS7` 主协议参考：`epoch=300`
- `wear_agg = mean`
- `no validation set`
- `LOCO`
- `Baseline` 或 `Baseline + KNN`

### 2.5 论文级特征消融核心阵列

最终只保留以下几条核心防线：

1. `Pure RMS7`
   - 轻量主线

2. `RMS7 + PTP7 (14D)`
   - 用于证明增加时域高阶幅值统计未必带来收益

3. `RMS7 + PTP7 + WAV7 (21D)`
   - 用于证明继续引入时频域扩展后，性能不一定提升，甚至可能退化

4. `Full-133D`
   - 行业基线
   - 用于说明高维全量统计特征在当前小样本跨工况设定下存在维度灾难与统计不稳定风险

### 2.6 输出物

本阶段必须产出一张最终论文级特征消融总表：

- 无 `NaN`
- 无旧协议残留
- 只保留最终协议结果
- 可直接放入 `Ablation Study on Feature Construction`

### 2.7 完成标准

当以下条件满足时，本阶段结束：

- `epoch=200` 探索数据不再混入终表
- 特征对比只保留同频结果
- `Full-133D / 14D / 21D / RMS7` 可在同一协议下直接横比

## 3. 第三阶段：论文叙事的乾坤大挪移

### 3.1 目标

既然实验已经证明 `Baseline + KNN` 优于 `TMA + KNN`，就必须立刻把论文主卖点从“TMA 是核心创新”切换为“轻量表征 + 隐空间检索”。

### 3.2 旧叙事的问题

旧叙事主要强调：

- 时间流形对齐
- TMA 作为主模块

但现在最终结果表明：

- `TMA` 单独增益有限
- `Baseline + KNN` 已经优于 `TMA + KNN`

因此，如果继续把 `TMA` 放在主创新点位置，证据会不够硬。

### 3.3 新主卖点 1：轻量物理表征

建议将第一主卖点改写为：

> **Physics-guided Lightweight Representation**

核心含义：

- 利用 `RMS` 这类具有明确物理含义的紧凑统计量构造输入
- 在保持可解释性的同时，避免高维全量统计特征带来的冗余与不稳定
- 证明 `TimerXL` 在轻量特征上更容易形成稳健泛化

### 3.4 新主卖点 2：隐空间几何检索

建议将第二主卖点改写为：

> **Latent Geometric Retrieval**

核心含义：

- 在极少样本、极端跨工况条件下，参数化回归头容易崩溃式过拟合
- `KNN` 不依赖额外参数拟合，而是直接利用 `TimerXL` 隐空间的几何邻近性进行检索修正
- 最终优势来自“非参数化检索兜底”，而不是更复杂的时间扭曲机制

### 3.5 TMA 的体面退场

`TMA` 不需要完全删除，但应降级为：

- 辅助机制探讨
- 或 Discussion 中的局限性分析

建议写法：

> 在当前统一协议与最终主线特征下，TMA 的独立贡献有限。其理念上强调时间流形对齐，但在缺乏目标域监督、噪声较大且跨工况偏移显著的条件下，时间步重排不一定总能保留原生的物理退化轨迹。相比之下，直接在隐空间上进行几何检索的 KNN 机制表现出更稳定的鲁棒性。

### 3.6 输出物

本阶段的输出不是实验，而是论文文本口径：

- 摘要改写
- 引言贡献点改写
- 方法章节主框架改写
- Discussion 中新增“TMA 局限性探讨”

## 6. 预期的论文终版结构

如果上述三阶段完成，论文最终会形成如下闭环：

1. **问题提出**
   - 小样本跨工况刀具磨损预测极易受到高维冗余与回归头过拟合影响

2. **解决方案**
   - 使用 `pure RMS7` 构造轻量、可解释输入
   - 使用 `TimerXL` 形成强时序隐表示
   - 使用 `KNN` 在隐空间进行非参数检索修正

3. **实验结论**
   - `pure RMS7` 优于更复杂的扩展特征
   - `Baseline + KNN` 优于 `TMA + KNN`
   - `Full-133D` 虽然 head-only 不稳，但其表示仍可被检索利用

4. **机制结论**
   - 轻量表征避免维度灾难
   - 非参数检索比时间步重排更稳健

# 讨论与局限草稿（中文）

## 1. 讨论

### 1.1 KNN-DRR 的主要价值在于修正结构性残差

当前正式结果表明，本文任务中的关键困难并不是整体均匀偏移，而是跨工况条件下未来磨损增量在 late-stage 被系统性低估。三折平均结果中，TimerXL head-only、TimerXL+TMA 和 TimerXL+TMA+KNN-DRR 的平均 MAE 分别为 `8.7711`、`6.1484` 和 `3.1784` um；在代表性困难场景中，head-only 的 late-stage MAE 为 `21.2180` um，而引入 KNN-DRR 后可降至 `5.7781` um。该结果说明，KNN-DRR 的作用并不是替代基础预测器，而是在 backbone 已经给出趋势估计的基础上，对未来磨损增量残差进行定向纠偏。

### 1.2 TMA 的作用应被理解为辅助增强，而非最终预测来源

TMA 的正式定位是训练期辅助增强。它首先改善跨工况时间节奏变化下的表示可比性，使检索空间更适合后续残差修正；随后 KNN-DRR 在这一更稳定的表示基础上完成主要性能提升。因此，本文的模块层级应保持为：TimerXL 提供 backbone，TMA 提供 auxiliary enhancement，KNN-DRR 完成 main residual correction。这样的叙述既与当前正式结果一致，也避免把 TMA 误写成独立的最终预测模块。

### 1.3 正式主线的贡献应聚焦于 few-shot cross-condition 场景

本文的主要结论成立于当前锁定的正式证据链：clean checkpoint、train-only 的 inner LOSO 参数选择、以及三折 LOCO 跨工况评估。在这一前提下，正式结果支持以下事实：一方面，跨工况退化速率差异会导致明显的 temporal-scale misalignment；另一方面，表示层增强与检索式残差修正的组合，能够稳定缓解该结构性偏差。讨论部分应围绕这一主线收束，而不应扩展到未经当前正式资产支持的更强泛化结论。

## 2. 局限性

### 2.1 当前验证仍限于 PHM2010 单数据集

本文目前的正式证据全部来自 PHM2010 few-shot cross-condition tool wear prediction 任务。虽然三折 LOCO 已覆盖该数据集中的全部跨工况组合，但这仍不足以支持对更广泛工业退化任务作出无条件泛化。因而，当前更稳妥的写法应是“在 PHM2010 及当前正式协议下结果稳定成立”，而不是直接外推到所有工业时序任务。

### 2.2 当前设定属于 history-aware wear forecasting

本文方法建立在当前正式任务设定之上，即测试阶段可使用最后一个已观测磨损值作为历史上下文的一部分。因此，当前结论对应的是 history-aware wear forecasting，而不是完全 sensor-only 的 blind prediction。对于仅有传感器信号、缺乏实时磨损观测的场景，仍需要额外的磨损状态估计步骤，这一点应在正文中主动说明。

### 2.3 检索式修正依赖训练库覆盖度

KNN-DRR 的优势在于直接使用训练域中的历史增量模式进行非参数纠偏，但这种优势仍依赖训练检索库对关键阶段样本的覆盖。当训练域的 late-stage 样本极少，或目标工况的退化轨迹明显偏离已观测历史时，检索式修正的有效性可能下降。因此，本文更适合被表述为“在当前 few-shot、但仍具备可用训练历史覆盖的场景中有效”，而不是完全脱离数据覆盖条件的普适解法。

### 2.4 可选增强证据尚未构成主文主张基础

当前正式主线已经足以支撑摘要、讨论和结论写作，但 backbone comparison 与 full133 三折比较仍属于可选增强项，不应被抬升为主结论证据。相应地，讨论与局限部分应主动说明：现阶段的正式结论主要基于 Table 1-6、Fig.1-Fig.7 以及对应正式 CSV，而不是依赖尚未补齐的增强性比较结果。

## 3. 小结

综合来看，本文当前最稳固的讨论边界是：TimerXL 提供基础表征，TMA 改善跨工况表示可比性，KNN-DRR 则直接修正 few-shot cross-condition 场景中的 late-stage residual bias。与此同时，单数据集验证、history-aware 设定、检索库覆盖依赖以及增强证据尚未完整，构成了本文在投稿版本中必须主动声明的局限边界。

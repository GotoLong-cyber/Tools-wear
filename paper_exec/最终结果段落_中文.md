# 最终结果段落（中文版）

## 主结果段落

在三折 LOCO 跨工况评估中，TimerXL head-only、TimerXL+TMA 和 TimerXL+TMA+KNN-DRR 的平均 MAE 分别为 `8.7711`、`6.1484` 和 `3.1784` um。该结果表明，单靠时序基础模型的表征能力不足以消除跨工况下的结构性预测偏差，而训练期时间尺度增强与检索残差修正分别在不同层级上提供了互补收益。具体而言，TMA 先改善了跨工况时间节奏变化下的表示可比性，而 KNN-DRR 进一步针对未来磨损增量的系统性低估进行了显式修正，从而形成了稳定的三折提升链条。

## 代表性困难场景结果段落

在代表性困难折上，head-only 模型的 late-stage MAE 达到 `21.2180` um，而在引入 KNN-DRR 后可降至 `5.7781` um。该结果说明，当前任务的关键误差并不是全局均匀偏移，而是后期未来磨损增量被系统性低估。换言之，KNN-DRR 的主要作用不是替代基础预测器，而是在基础预测已经给出趋势的前提下，通过历史相似轨迹对增量残差进行定向纠偏。

## 机制解释段落

TMA 的作用应被理解为检索可用性增强，而非独立的最终预测模块。从三折结果看，TMA 将平均 MAE 从 `8.7711` 降至 `6.1484` um，说明盲式 train-time multi-stride augmentation 可以有效缓解跨工况退化节奏差异带来的时间尺度错位。在此基础上，采用 inner LOSO train-only validation 选出的全局固定参数 `k=10, beta=0.7, late_q=0.0` 后，KNN-DRR 将三折平均 MAE 进一步降至 `3.1784` um，说明更接近真实 cross-condition 任务的参数选择协议能够显著提升正式主结果的可信度与性能表现。

## 参数选择协议段落

本文所有正式 retrieval 结果均不基于 outer test fold 选参。我们在每个 outer fold 的源工况内部执行 inner leave-one-source-out validation：使用一个源工况构建 retrieval library，另一个源工况作为验证目标，随后交换两者角色，并在全部 inner splits 上平均验证误差，最终确定全局固定参数。该协议比同源 train/val 更能保留 cross-condition gap 与 wear-rate mismatch，因此比简单固定经验参数更符合工业跨工况预测的实际设置。

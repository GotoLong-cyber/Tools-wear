# 最终图注

- `Fig.1 / Fig01_wear_progression_three_conditions`
  图注：PHM2010 三个工况下的磨损演化曲线及其时间尺度错位现象。
  要点：不同工况在相同切削步下并不对应相同磨损阶段，跨工况预测首先面临时间尺度错位。

- `Fig.2 / Fig02_framework_overview`
  图注：本文方法框架，包括 TimerXL backbone、TMA 训练期增强和 KNN-DRR 残差检索修正。
  要点：TimerXL 提供基础表示，TMA 提升表示可比性，KNN-DRR 是主创新模块，负责未来磨损增量残差修正。

- `Fig.3 / Fig03_hardest_fold_progressive_compare`
  图注：代表性困难折上从 TimerXL head-only 到 TMA 再到 KNN-DRR 的逐步预测改进曲线。
  要点：晚期低估是结构性误差，KNN-DRR 对该误差的修正最为明显。

- `Fig.4 / Fig04_three_fold_curve_compare`
  图注：三折 LOCO 评估中，TimerXL head-only 与 KNN-DRR 的全曲线对比结果。
  要点：检索增强收益在三折上均存在，且在代表性困难场景上更为显著。

- `Fig.5 / Fig05_late_stage_residual_correction`
  图注：代表性困难折的分段 MAE 与 mean residual 对比。
  要点：KNN-DRR 主要修正的是 late-stage residual bias，而不是对整条曲线进行均匀平移。

- `Fig.6 / Fig06_retrieval_quality_before_after_tma`
  图注：TMA 前后检索质量指标对比，包括距离、late-library 命中率与 late-stage delta error。
  要点：TMA 改善的是 retrieval usefulness，而不是要求所有距离指标都单调变优。

- `Fig.7 / Fig07_protocol_boundary`
  图注：train/validation/test 的协议边界示意图。
  要点：正式主结果严格遵守 train-only library construction、inner LOSO 选参和 outer test 单次评估。

- `Fig.A1 / FigA1_inner_loso_sensitivity`
  图注：inner LOSO train-only validation 下的参数敏感性附图。
  要点：`beta=0.7, late_q=0.0` 在 inner validation 上表现稳定优于其邻近配置，因此最终全局固定参数不是拍脑袋选取。

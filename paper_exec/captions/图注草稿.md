# 图注草稿

- 文件名: `Fig01_wear_progression_three_conditions`
- 目标图号: `Fig.1`
- 图注: PHM2010 三个工况下的磨损演化曲线及其时间尺度错位现象。
- 要点: C6 的磨损推进更快，因此相同时间索引并不对应相同磨损阶段。

- 文件名: `Fig02_framework_overview`
- 目标图号: `Fig.2`
- 图注: zero-leakage 协议下 TimerXL backbone、TMA 与 KNN-DRR 的整体框架。
- 要点: TimerXL 提供表示，TMA 提升可比性，KNN-DRR 负责主要残差修正。

- 文件名: `Fig03_hardest_fold_progressive_compare`
- 目标图号: `Fig.3`
- 图注: 代表性困难折上从 TimerXL head-only 到 TMA 再到 KNN-DRR 的逐步预测改进曲线。
- 要点: late-stage 低估是结构性误差，在引入基于检索的残差融合后修正最明显。

- 文件名: `Fig04_three_fold_curve_compare`
- 目标图号: `Fig.4`
- 图注: 三折 LOCO 评估中 TimerXL head-only 与 KNN-DRR 的预测曲线对比。
- 要点: 检索增强模型在三折上均有提升，在代表性困难场景上的收益最大。

- 文件名: `Fig05_late_stage_residual_correction`
- 目标图号: `Fig.5`
- 图注: 代表性困难折上的分阶段 MAE 与 residual 对比。
- 要点: KNN-DRR 主要修正的是 late-stage residual bias，而不是对整条曲线做均匀平移。

- 文件名: `Fig06_retrieval_quality_before_after_tma`
- 目标图号: `Fig.6`
- 图注: 三折中 TMA 前后的检索质量对比。
- 要点: 即使原始 cosine 距离并非每折都单调下降，TMA 仍主要改善 late-stage 的检索可用性与残差修正质量。

- 文件名: `Fig07_protocol_boundary`
- 目标图号: `Fig.7`
- 图注: train/validation/test 分离的协议边界示意图。
- 要点: 正式证据必须遵守 train-only library 构建和测试前固定超参数的原则。

# 下一步总执行计划

更新时间：2026-04-01

## A. 当前项目阶段判断

当前项目处于以下三项的组合阶段：

1. 证据整合收口
2. 论文资产锁定
3. 正文写作推进

判断依据：

- 主线 clean 证据链已经闭合，正式主结果已锁定为 inner LOSO 选参后的三折结果。
- `paper_exec` 主图、主表、主结果描述已经存在，但仍需做最终一致性复核，确保写作阶段不再引用旧结果源。
- 当前主线阻塞已不再是协议或训练，而是“把现有正式证据准确地写进论文，并补齐投稿前最后一层一致性检查”。

## B. 单一线性优先级序列

### Step 1. 冻结正式主结果源

目标：

- 把当前正式主结果源明确锁定为 `results/20260401_RetrievalV21_innerlososelect`
- 明确旧结果只保留为参考，不再进入正文主表主图

所需输入：

- `paper_exec清单.md`
- `paper_exec/论文资产看板.md`
- `paper_exec/manifests/最终主线刷新报告.md`
- `results/20260401_RetrievalV21_innerlososelect/*`

完成标准：

- 所有主线说明文件都只认 `inner LOSO + clean checkpoint` 这一套正式结果源
- `20260401_RetrievalV21_cleanformal`、`20260401_RetrievalV21_trainonlyselect`、`20260330_RetrievalV21_fixedcfg_rerun` 都被明确标成参考或弃用

若跳过风险：

- 正文、图表、口头汇报可能混用旧结果与新结果，直接破坏论文可信度

完成后应更新：

- `paper_exec清单.md`
- `paper_exec/论文资产看板.md`
- `paper_exec/manifests/最终主线刷新报告.md`

### Step 2. 完成正式证据闸门核验

目标：

- 对主表、主图、主 CSV、caption、结果段落逐项判定是否可以作为正式证据进入论文正文

所需输入：

- `paper_exec/csv/main_results_3fold.csv`
- `paper_exec/csv/hardest_scenario_results.csv`
- `paper_exec/csv/retrieval_ablation.csv`
- `paper_exec/csv/stage_residual_compare.csv`
- `paper_exec/csv/retrieval_quality_before_after_a2.csv`
- `paper_exec/csv/tma_ablation.csv`
- `paper_exec/figures/Fig01-Fig07`
- `paper_exec/captions/*.md`
- `paper_exec/结果片段_中文.md`
- `paper_exec/结果片段_英文.md`

完成标准：

- 每个主线资产都满足 5 个条件：
  1. 来自 clean protocol
  2. 使用 inner LOSO 选出的固定参数
  3. 与 execution log 和 manifest 记录一致
  4. 不混入弃用结果目录
  5. 在 `paper_exec` 中已有唯一正式版本

若跳过风险：

- 论文写作时会把“已生成”误当成“已正式可用”，导致 claim 边界失真

完成后应更新：

- `paper_exec/正式证据门禁.md`
- `paper_exec/论文资产看板.md`

### Step 3. 锁定正文将使用的图表集合

目标：

- 明确正文只使用哪几张图、哪几张表
- 明确哪些内容转入附录或暂不使用

所需输入：

- `paper_exec/论文资产看板.md`
- `paper_exec/投稿就绪报告.md`
- `paper_exec/tables/*.md`
- `paper_exec/figures/*.pdf`

完成标准：

- 主文固定为：
  - 图：Fig.1–Fig.7
  - 表：Table 1–6
- `Table 7 / Table 8 / Table A2 / Fig.A1` 被明确标记为附录或增强证据

若跳过风险：

- 写正文时会不断摇摆，导致方法节和实验节结构反复改动

完成后应更新：

- `paper_exec/稿件完成顺序.md`
- `paper_exec/投稿就绪报告.md`

### Step 4. 先写方法节，再写实验节

目标：

- 用当前正式证据链完成论文最关键的两节正文

所需输入：

- `method_skeleton.md`
- `experiment_skeleton.md`
- `paper_exec/最终结果段落_中文.md`
- `paper_exec/最终结果段落_英文.md`
- `paper_exec/论文主张边界.md`
- `paper_exec/captions/图注定稿.md`
- `paper_exec/captions/表注定稿.md`

完成标准：

- 方法节必须明确写出：
  - TimerXL 是 backbone
  - TMA 是辅助训练期增强
  - KNN-DRR 是主创新
  - inner LOSO 是 train-only 的参数选择协议
- 实验节必须完整写出：
  - 三折主结果
  - 代表性困难场景
  - retrieval ablation
  - TMA 作用解释

若跳过风险：

- 即使结果已齐，也无法快速收口为可投稿文稿

完成后应更新：

- 论文正文草稿
- `paper_exec/最终结果段落_中文.md`
- `paper_exec/最终结果段落_英文.md`

### Step 5. 完成图表-叙事-主张一致性检查

目标：

- 确保正文表述、图注、表注、结果段落、主张边界完全一致

所需输入：

- `paper_exec/论文主张边界.md`
- `paper_exec/captions/图注定稿.md`
- `paper_exec/captions/表注定稿.md`
- 方法节草稿
- 实验节草稿

完成标准：

- 不再出现以下冲突：
  - 把 TimerXL 写成主创新
  - 把 hardest scenario 写成设计先验
  - 把 KNN-DRR 写成替代 backbone
  - 把 inner LOSO 写成 outer test 选参

若跳过风险：

- 审稿时最容易被抓住“叙事和证据不一致”

完成后应更新：

- `paper_exec/论文主张边界.md`
- 论文正文草稿

### Step 6. 清理参考文献与实现细节引用

目标：

- 在不新增实验的前提下，补齐写作所需的相关工作、数据集协议、实现细节引用位

所需输入：

- 现有摘要/引言/方法/实验骨架
- 当前论文草稿

完成标准：

- 相关工作引用位完整
- 数据集、LOCO 协议、few-shot、cross-condition、foundation model、retrieval correction 的引用位齐全
- 不再出现“待补引用”或“后面再查文献”这种占位

若跳过风险：

- 稿件结构完整但参考文献不足，投稿前会卡住

完成后应更新：

- 论文正文草稿
- 参考文献库

### Step 7. 生成最终投稿前风险清单

目标：

- 在真正投稿前，把剩余风险压缩到一个短清单

所需输入：

- 全部正式图表
- 全部正文草稿
- `paper_exec/论文主张边界.md`
- `paper_exec/投稿就绪报告.md`

完成标准：

- 风险清单只保留有限条目：
  - 可选 baseline 未补齐
  - Full133 非完整
  - 附录是否加入 sensitivity
- 不再存在主线协议或主结果来源不清的问题

若跳过风险：

- 投稿前最后一轮自查容易遗漏边界问题

完成后应更新：

- `paper_exec/投稿就绪报告.md`
- 最终投稿 checklist

## C. 当前最小闭环

从现在开始，最小、最有效的顺序就是：

1. 冻结正式结果源  
2. 做正式证据闸门核验  
3. 锁定正文图表集合  
4. 写方法节  
5. 写实验节  
6. 做图表-叙事一致性检查  
7. 清理参考文献  
8. 生成最终投稿风险清单

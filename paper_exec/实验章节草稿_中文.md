# 4 实验设计与结果分析

## 4.1 数据集与数据预处理（Dataset and Preprocessing）

本文在 PHM2010 公开刀具磨损数据集上验证所提方法。该数据集由 Röders Tech RFM760 高速数控加工中心采集，工件材料为 Inconel 718，刀具为直径 6 mm 的三刃硬质合金球头铣刀。实验采用干式铣削，并通过七路传感器在线记录加工过程中的多模态响应，包括三轴切削力、三轴振动以及一路声发射（AE）信号，采样频率为 50 kHz。数据集中的磨损标签由三条切削刃分别测量得到，当前主文整理稿默认沿用三刃后刀面磨损中的最大值作为该 pass 的磨损标签；与此同时，我们也补充完成了“三刃均值标签”口径下的 retrieval 主链三折重跑，用于检查标签聚合定义对结果的影响，相关结果见第 4.7 节。

PHM2010 中带标签的典型工况为 `c1`、`c4` 和 `c6`。三者对应不同的切削参数组合，因此磨损推进节奏并不一致。就轨迹层面而言，这种工况差异会直接表现为明显的退化速率差别：在相同切削步数下，`c6` 往往更早进入高磨损阶段，而 `c4` 的磨损推进相对平缓。以训练域 `c1+c4` 拟合得到的阈值为参照，`c6` 到达 `train_q75` 与 `train_q90` 的时间点分别比 `c1` 早 30 和 61 个 cut，比 `c4` 早 58 和 44 个 cut。该现象说明，本文面对的困难并不只是一般的分布偏移，而是带有明确退化节奏失配的跨工况预测问题。

在数据表示层面，本文并未直接把 50 kHz 原始波形作为输入，而是先按 pass 聚合为低维统计特征，再进行时序建模。当前 formal-ready 主线采用 9 维输入特征：七个通道各自的 RMS 特征（`Feat_3, Feat_7, Feat_11, Feat_15, Feat_19, Feat_23, Feat_27`），加上通道 1 的幅值范围特征 `Feat_4`，以及正交补充的频谱复杂度特征 `SPECTRAL_ENTROPY_CH1`。该设计兼顾了多传感器幅值信息与局部谱结构信息，同时保持输入维度紧凑，便于在小样本跨工况设置下稳定训练。

## 4.2 跨工况少样本评测协议（Cross-Condition Few-Shot Evaluation Protocol）

本文采用严格的三折 leave-one-condition-out（LOCO）评测协议。具体而言，fold1 设定为 `c1,c4 -> c6`，fold2 为 `c4,c6 -> c1`，fold3 为 `c1,c6 -> c4`。在每个外层折中，两个源工况仅用于训练与内部验证，目标工况在模型训练、特征选择、检索库构建和超参数确定阶段均被完全封存，只在最终评测时使用一次。该协议使每一折都保留了真实的 cross-condition gap，因此比同工况随机划分更能反映实际部署中的泛化难度。

本文的 few-shot 含义不是“测试工况提供少量标签用于再训练”，而是“模型只能依赖极少量源工况历史轨迹学习跨工况规律，而测试工况在推理时不提供任何未来标签用于微调”。更具体地说，本文任务属于 history-aware wear forecasting：给定前 96 个历史时间步，预测未来 16 个时间步的磨损量。测试阶段允许访问目标工况的历史观测窗口，但不允许使用其未来真实磨损值进行任何形式的参数更新、时间步自适应对齐或检索库扩充。

为了进一步堵住“outer test 选参”带来的数据泄露质疑，本文在每个外层折内部采用 inner leave-one-source-out（inner LOSO）训练内选参协议。设两个源工况分别为 `A` 和 `B`，则先用 `A` 构建检索库并用 `B` 作为验证目标，再交换角色重复一次；随后在全部 inner task 上聚合验证误差，从候选网格 `k in {3,5,10}`、`beta in {0.3,0.5,0.7}`、`late_q in {0.0,0.8}` 中确定唯一的全局固定配置。最终 formal 主线锁定 `k=10, beta=0.7, late_q=0.0`，并在三折 outer test 上一次性评估。需要强调的是，TMA 仅作用于训练阶段的数据增强，验证与测试阶段均固定 stride=1，不执行任何针对测试工况的自适应对齐。

## 4.3 对比模型与评价指标（Baselines and Evaluation Metrics）

考虑到本文的核心目标是验证“TimerXL backbone、TMA auxiliary enhancement 与 KNN-DRR main correction”这一级联结构是否成立，主文中的对比以层级化内部基线为主，而非堆砌大量异构模型。正式主线包含三组核心对比：`TimerXL head-only`，用于评估预训练时序骨干在跨工况小样本下的基础能力；`TimerXL + TMA`，用于验证训练期时间尺度增强对表示可比性的贡献；`TimerXL + TMA + KNN-DRR`，作为本文完整方法，用于评估检索式增量残差修正带来的最终收益。此外，为了进一步拆解检索模块的作用方式，本文还报告 `knn-only`、`delta-knn-only` 与 `delta-blend` 三种检索模式消融。需要说明的是，其他异构骨干网络对比目前仍属于不完整增强资产，因此不作为主文主结论的支撑证据。

评价指标方面，本文主指标采用整条预测曲线上的 MAE 与 RMSE，并统一报告 `fullcurve_raw` 口径，以避免不同后处理方式造成的结果混淆。除此之外，针对 hardest fold，本文进一步进行 early/mid/late 三阶段误差分解，统计各阶段的 MAE、RMSE、mean residual 与低估比例，用于回答“模型到底错在何处”和“KNN-DRR 具体修复了什么结构性偏差”这两个核心问题。换言之，主结果表负责说明整体性能提升，阶段分解与检索消融负责解释这些提升为何产生。

## 4.4 跨工况预测结果分析（Cross-Condition Prediction Results Analysis）

表 2 给出了三折 LOCO 下的正式主结果。可以看到，`TimerXL head-only`、`TimerXL + TMA` 和 `TimerXL + TMA + KNN-DRR` 的平均 MAE 分别为 `8.7711`、`6.1484` 和 `3.1784` μm，对应平均 RMSE 分别为 `11.9747`、`9.4630` 和 `3.9763` μm。相较于基础骨干，单独引入 TMA 可将平均 MAE 降低 `29.9%`；在此基础上继续加入 KNN-DRR 后，平均 MAE 相对 `TimerXL + TMA` 再下降 `48.3%`，相对 `TimerXL head-only` 的总降幅达到 `63.8%`。这一结果说明，仅依赖预训练时序基础模型并不足以消除跨工况条件下的结构性误差，而训练期可比性增强与推理期残差修正提供了明确且互补的两段收益。

从逐折结果看，`c1,c4 -> c6` 是最能暴露问题的代表性困难场景。在该折中，整体 MAE 从 `13.3984` μm 下降到 `9.2761` μm，再进一步下降到 `3.2065` μm，完整方法相较于基础骨干的降幅达到 `76.1%`。对应的预测曲线图应重点展示两点：其一，基础模型在早中期通常还能跟随总体趋势，但在晚期高磨损区间会出现明显“贴不住真实曲线”的现象；其二，引入 KNN-DRR 后，模型不再只给出保守的平均化预测，而是能够在晚期随真实曲线一同抬升。这也是本文建议在主文中保留 hardest fold 预测曲线图的原因，因为它比平均指标更直观地暴露了 late-stage underestimation 的结构性特征。

表 4 的阶段误差分解进一步支持上述判断。以 hardest fold 上的检索骨干 `head-only` 为例，early、mid 与 late 阶段的 MAE 分别为 `3.6325`、`2.8863` 和 `21.2180` μm，其中 late-stage 误差远高于前两个阶段；对应的 mean residual 为 `-20.4768` μm，低估比例达到 `0.8493`。这说明当前任务的主导误差并不是全局平移型偏差，而是晚期未来磨损增量被系统性低估。进一步引入 `delta-blend@k10_b07` 后，late-stage MAE 降至 `5.7781` μm，较 head-only 下降 `72.8%`，mean residual 同时从强负偏压缩至 `-5.7781` μm。换言之，本文方法的最大收益集中在最困难、也最关键的晚期高磨损区间。

## 4.5 关键模块消融实验（Ablation Study of Key Components）

本文的模块消融分为两层。第一层是主链消融，即 `No TMA (TimerXL head-only)`、`+TMA only` 与 `+TMA + KNN-DRR` 的层级对比，如表 6 所示。三折平均结果表明，TMA 本身已能带来稳定收益，但真正决定最终上限的是其后接入的残差修正模块。以 hardest fold 为例，整体 MAE 从 `13.3984` 降至 `9.2761`，说明 TMA 可以先缓解部分时间尺度失配；进一步引入 KNN-DRR 后，MAE 继续降至 `3.2065`。若只看 late-stage，更能看出模块分工差异：三组方法在 fold1 上的 late-stage MAE 分别为 `32.4302`、`25.6950` 和 `5.7781` μm，说明 TMA 主要负责“先把表示空间变得更可比”，而 KNN-DRR 才是对系统性低估进行强修复的关键。

第二层是检索模式消融，即表 5 中的 `head-only`、`knn-only@k10`、`delta-knn-only@k10` 和 `delta-blend@k10_b07`。该组结果的核心结论有三点。第一，`knn-only` 在 fold1 上的 MAE 高达 `25.8334` μm，明显失败，这说明纯检索无法替代时序骨干提供的全局趋势建模。第二，`delta-knn-only` 在三个 fold 上都表现很强，表明“预测未来增量残差”比“直接投票未来绝对值”更符合当前任务的误差结构。第三，`delta-blend` 虽然在个别折上略逊于 `delta-knn-only`，但整体更稳健，也更符合“骨干给趋势、检索做纠偏”的方法定位，因此被确定为正式主方案。换言之，本文并不是证明“检索越强越好”，而是在证明“只有把检索限制为增量残差修正，才能在跨工况 few-shot 设置下稳定发挥作用”。

## 4.6 核心模块深度分析（Deep Analysis of KNN-DRR and TMA）

### 4.6.1 KNN-DRR 机制分析

KNN-DRR 的核心不是直接从历史库中“抄答案”，而是从训练专属库中召回与当前测试窗口最相似的历史片段，并估计其未来磨损增量 $\Delta \mathbf{y}$ 相对基础预测仍存在多少残差。于是，最终输出不是由检索值直接替代得到，而是通过残差混合完成修正：

$$
\hat{\mathbf{y}}_{\text{final}} = \hat{\mathbf{y}}_{\text{backbone}} + \beta \cdot \mathbf{r}_{\text{knn}}.
$$

这里的 $\mathbf{r}_{\text{knn}}$ 表示基于相似历史轨迹估计得到的未来增量残差。该设计有两个直接好处：一方面，它保留了 TimerXL 对整体趋势和局部时序依赖的建模能力；另一方面，它把检索模块的职责限制在最容易出错的“晚期未来增量低估”上，而不是让纯近邻投票主导整条预测曲线。表 5 的消融已经说明，`knn-only` 明显不稳，而 `delta-knn-only` 与 `delta-blend` 显著更强，正是因为后者显式针对增量残差而非绝对值进行修正。

参数 $\beta$ 的作用可以理解为“修正强度”的控制系数。若 $\beta$ 过小，检索残差难以有效纠正骨干的晚期保守偏差；若 $\beta$ 过大，则可能使最终预测过度依赖邻居而牺牲曲线整体稳定性。本文通过 inner LOSO train-only 验证在源工况内部搜索 `beta in {0.3, 0.5, 0.7}`，最终选择 `beta=0.7`。从敏感性结果看，`beta=0.7` 与 `late_q=0.0` 组成的配置形成最优稳定区域，而 `beta=0.5` 与 `beta=0.3` 的平均 inner validation MAE 分别退化到 `3.8877` 和 `4.5299`。这说明在当前任务上，较强但不过度的残差混合更有利于修复 late-stage underestimation。

### 4.6.2 TMA 机制分析

TMA 的作用不应被表述为“测试时对齐目标工况”，而应被理解为一种仅发生在训练阶段的时间尺度增强。它通过在源工况窗口上引入多步幅采样，使模型在参数更新过程中同时见到原始时间分辨率与压缩时间分辨率下的退化轨迹，从而减弱“固定时间索引等价于固定磨损阶段”的错误先验。由于验证与测试阶段均固定 stride=1，TMA 并不会利用任何来自目标工况的未来信息，因此不会破坏 zero-leakage 边界。

在证据层面，TMA 的收益首先体现在主链消融中：三折平均 MAE 从 `8.7711` 降至 `6.1484`，说明其对跨工况表示可比性的改善具有普适性。更细粒度地看，retrieval quality 诊断表明，TMA 虽不一定总能让最近邻距离指标更小，但它能更稳定地降低基于检索得到的 late-stage delta error。例如在 fold2 与 fold3 中，`delta_error_mae_late` 由 baseline 的 `4.8224` 和 `3.7670` 分别下降到 TMA 版本的 `3.7717` 和 `3.5369`。这说明 TMA 的收益不应被简单理解为“几何距离变近”，而更应被理解为“经过训练期时间尺度增强后，表示空间对后续残差检索更有用”。

如果篇幅允许，本节还可以在附录中补充一张表示空间可视化图，例如 t-SNE 或 UMAP，用于直观展示 stride=2 增强后源工况与目标工况窗口在高磨损区间的混合程度变化。但从当前 formal 证据链看，即便不额外引入新的可视化实验，TMA 对下游检索可用性和最终误差下降的贡献也已经可以由表 2、表 5、表 6 以及 retrieval quality 诊断共同支撑。

## 4.7 三刃均值标签的补充重跑结果

考虑到 PHM2010 每个 pass 同时提供三个切削刃的磨损测量值，而标签聚合口径可能影响模型训练目标，我们补充进行了“三刃均值”标签定义下的 retrieval 主链重跑。该补充实验保持与 formal retrieval 主线一致的训练与评测边界：仍采用三折 LOCO，仍使用 `TimerXL retrieval-backbone + TMA + KNN-DRR` 的级联结构，仍通过 inner-LOSO train-only 协议固定 `k=10, beta=0.7, late_q=0.0`，唯一变化是将每个 pass 的磨损标签由三刃最大值改为三刃平均值。

| 目标工况 | head-only MAE | head-only RMSE | delta-blend MAE | delta-blend RMSE |
|----------|---------------|----------------|-----------------|------------------|
| C6（fold1） | 6.6100 | 9.5561 | **2.7879** | **3.3697** |
| C1（fold2） | 4.5268 | 5.6774 | **2.9303** | **3.6481** |
| C4（fold3） | 4.6019 | 5.3914 | **3.0869** | **3.7043** |
| 三折平均 | 5.2462 | 6.8749 | **2.9350** | **3.5741** |

从该补充结果可以看到，在三刃均值标签口径下，retrieval 骨干 `head-only` 的三折平均 MAE 为 `5.2462` μm，而完整 `delta-blend@k10_b07` 的三折平均 MAE 进一步下降到 `2.9350` μm，说明即使在标签定义改为平均值后，KNN-DRR 依然能够稳定提供显著增益。具体到 hardest fold `c1,c4 -> c6`，均值标签下 `head-only` 与 `delta-blend` 的 MAE 分别为 `6.6100` 和 `2.7879` μm，晚期低估仍然是主要误差来源，但其绝对误差水平整体低于最大值标签版本。

需要强调的是，这一补充实验目前仅完成了 retrieval 主链重跑，因此它更适合被写作“标签聚合敏感性补充证据”，而不宜直接替换主文当前基于最大值口径整理的所有主表数字。若后续决定将全文正式标签口径统一切换为“三刃均值”，则还需要同步补齐基础骨干与 `TimerXL + TMA` 两组主链结果，并据此重写主表与摘要中的平均指标。

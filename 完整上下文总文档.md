## 2. 项目核心任务定义

### 2.1 任务对象

项目聚焦于 `PHM2010` 刀具磨损预测任务，主要研究跨工况、跨刀具条件下的多步磨损预测。

### 2.2 当前统一任务设定

当前正式设定是：

1. 输入：过去 `96` 个时间步的历史观测；
2. 输出：未来 `16` 个时间步的磨损预测；
3. 单位：最终以反归一化后的 `wear (um)` 计算 `MAE / RMSE`；
4. 当前模型输入并不只是传感器特征，还包含历史磨损值；
5. 因此当前任务更准确地定义为：
   - `history-aware wear forecasting`

### 2.3 当前不是的任务

当前正式主线**不是**：

1. 严格 `sensor-only online prediction`
2. 不读取历史磨损值的纯传感器预测
3. 测试阶段完全不接触测试窗口历史状态的设定

这一点必须明确，因为后面的 `Retrieval V2.1` 直接建立在这个任务边界之上。

## 3. 数据与输入口径

### 3.1 数据集与刀具

当前主实验围绕 `c1 / c4 / c6` 三把刀展开，形成三折 leave-one-tool-out 跨工况验证。

### 3.2 三折定义

当前正式三折定义固定为：

1. `fold1`: `train=c1,c4`, `test=c6`
2. `fold2`: `train=c4,c6`, `test=c1`
3. `fold3`: `train=c1,c6`, `test=c4`

这是项目里后续所有结果表、曲线图、检索实验、敏感性分析的统一折别口径。

### 3.3 输入列构成

根据当前数据管线，模型输入是：

1. 当前保留的传感器特征列
2. 最后一列磨损值 `wear`

也就是说，当前模型测试时会读到：

1. 前 `96` 步历史特征
2. 前 `96` 步历史磨损值

这也是后续与 `Gemini` 讨论时“是否使用测试集数据”的关键边界来源。

### 3.4 当前统一指标口径

当前主要汇报指标为：

1. `window` 级指标
2. `fullcurve`
3. `fullcurve_raw`

其中后期项目主线主要使用：

- **`fullcurve_raw MAE / RMSE`**

这一定义为：

1. 先对每个测试窗口预测未来 `16` 步；
2. 将所有窗口输出重建成整条预测磨损曲线；
3. 再和真实原始磨损曲线逐点比较；
4. 在有效预测区间上计算 `MAE / RMSE`

因此，当前汇报的主结果不是单窗口平均误差，而是整条磨损曲线重建后的误差。

## 4. 零泄露与实验协议

### 4.1 当前项目对“零泄露”的正式定义

从 [实施计划.md](/home/jyc23/zml25/Timer/FeatureExact3.10/FeatureTest/OpenLTM/实施计划.md) 与 [实施计划2.md](/home/jyc23/zml25/Timer/FeatureExact3.10/FeatureTest/OpenLTM/实施计划2.md) 固化下来的正式定义是：

泄露的判断不是“有没有用到 `c6`”，而是：

1. 当前折测试域是否参与任何拟合步骤；
2. 标准化、特征选择、模型选择、超参数确定是否只基于训练域；
3. 测试域是否只用于最终盲测与离线分析。

### 4.2 当前折内的硬约束

每个 fold 都要求：

1. 特征提取规则若涉及可学习参数，只能在训练域拟合；
2. 特征选择与打分只能在训练域完成；
3. 标准化参数只能由训练域拟合；
4. 模型选择、早停与超参不能参考测试域；
5. 测试域只做最终盲测；
6. 盲测后可以做误差分析，但不能回流重调模型

### 4.3 当前协议与“严格在线预测”的张力

这个项目里最需要单独强调的地方是：

1. 当前协议在测试时允许读取测试窗口历史输入；
2. 历史输入中包含历史磨损值；
3. `Retrieval V2.1` 还使用测试窗口最后一个已观测磨损值作为锚点；
4. 但不使用测试未来真实标签参与建库、检索、加权或融合；
5. 测试未来真值只用于最终离线评估

因此，当前正式最优方案属于：

- `history-aware wear forecasting`

而不是：

- `sensor-only online prediction`

这既是方法边界，也是后续答辩和论文写作中必须主动交代的核心点。

## 8. 导师沟通后的问题重构

根据 [2026.3.20沟通.txt](/home/jyc23/zml25/Timer/FeatureExact3.10/FeatureTest/OpenLTM/2026.3.20沟通.txt) 与 [2026.3.26沟通.txt](/home/jyc23/zml25/Timer/FeatureExact3.10/FeatureTest/2026.3.26沟通.txt)，导师反复强调的核心问题可以压缩成四句：

这些意见后面分别落地成：

1. `A1/A2` 时间尺度实验

4. 论文要强调“给别人什么启示”，而不是只强调“最终数值更优”。

## 9. A1：时间尺度错位的诊断证据

### 9.1 A1 的目的

`A1` 不是为了直接提升模型，而是为了证明：

- `C6` 在磨损过程上和 `C1/C4` 存在明显时间尺度错位

### 9.2 A1 的结论

通过训练域分位数和阶段性诊断，已经得到稳定判断：

1. `C6` 更早进入高磨损阶段；
2. 同样时间步下，`C6` 的磨损曲线更陡；
3. 因而固定时间窗口并不等价于固定磨损阶段；
4. 这直接支撑了后续 `A2` 的必要性

更具体地，`A1` 使用训练域 `c1/c4` 拟合阈值后，对三把刀做“首次跨阈值 cut”比较，得到了更强的事后证据：

1. 训练域 `q75` 阈值为 `130.7415 um`，`c1/c4/c6` 首次到达 cut 分别为 `223 / 251 / 193`；
2. 训练域 `q90` 阈值为 `162.0820 um`，`c1/c4/c6` 首次到达 cut 分别为 `293 / 276 / 232`；
3. `c6` 在高阈值处显著更早跨入高磨损区，说明其 wear progression pace 明显快于训练域；
4. 这些 crossing 指标来自 [a1_summary.md](/home/jyc23/zml25/Timer/FeatureExact3.10/FeatureTest/OpenLTM/feature_alignment_diagnosis/outputs/A1_time_scale_20260323_1539/a1_summary.md)，属于 **post-hoc evidence**，用于解释项目为何从“枚举特征”转向“时间尺度问题”，**does not flow back into training decisions**。

### 9.3 这一步在项目中的地位

`A1` 是整个主线从“特征工程”转向“时间对齐”的转折点。

## 10. A2：时间压缩增强主线

### 10.1 A2 的真正含义

`A2` 不是公开方法缩写，而是项目内部方法编号。

当前定版定义是：

- `A2 = S2-only train-time time compression augmentation`

中文可表述为：

- **仅使用 `stride=2` 的训练时时间压缩增强**

当前正式实现名：

- `A2-Random-S2`

### 10.2 旧版 A2

旧版 `A2_stagealign` 曾经在 `fold1` 上表现很好，但后来经 review 认定存在方法学风险：

1. 训练时对时间轴做了更重的全局重参数化；
2. 容易引入语义不一致；
3. 只保留为“诊断性证据”，不再作为正式主结论

### 10.3 新版 A2 动态步长增强

正式版 A2 的关键思想是：

1. 不改测试域时间轴；
2. 只在训练时构造部分 `stride=2` 压缩窗口；
3. 让模型看到“更快演化”的训练样本；
4. 用训练时增强去缓解跨工况时间节奏失配

这里必须强调一个写作边界：

1. `A2` 不能表述为“根据 `C6` 更快，所以反向去对齐 `C6`”；
2. 更安全且与实现一致的表述是：
   - 在跨工况任务中，不同刀具可能存在不同的 wear progression pace；
   - 固定时间步构窗会导致“相同历史长度不等于相同磨损阶段”；
   - 因此在训练阶段引入 multi-stride temporal augmentation，以提升模型对演化速率变化的鲁棒性；
3. 这是一种 **train-time augmentation**，不是基于测试未来标签或测试曲线的反向定制。

### 10.4 A2 三轮消融后的稳定结论

已稳定确认：

1. `stride=2` 有效；
2. `stride=3` 没贡献；
3. `random-S2` 并不比 `slope-guided` 差；
4. 单独做 late-stage 重采样/重加权不能复现收益；
5. 因此 A2 的核心有效因素是：
   - **训练时的 `stride=2` 时间压缩增强本身**

结合 [A2_dynstride图解.md](/home/jyc23/zml25/Timer/FeatureExact3.10/FeatureTest/OpenLTM/feature_alignment_diagnosis/outputs/A2_dynstride_diagnostics_20260323_1743/A2_dynstride图解.md)，当前还可以更明确地补充三点：

1. `val/test` 始终固定为 `stride=1`，没有对测试域做任何重采样；
2. 真正大量生效的是 `stride=2`，`stride=3` 在三折里几乎没有形成稳定覆盖，因此不是正式主线的一部分；
3. `A2` 的项目作用不是“彻底解决 hardest fold”，而是把问题从“全段都不行”收缩成“主要后段不行”，为后续 `Retrieval V2.1` 留出更明确的修复对象。

### 10.5 A2 在项目中的地位

`A2` 是当前主线的第一关键创新点：

1. 它正面回答了导师提出的“时间尺度不对齐”问题；
2. 三折均优于 `RMS7 + Feat_4` 主基线；
3. 说明时间尺度错位确实是主要瓶颈之一。

这一步说明：

1. 不是继续横向加普通特征就能解决 hardest fold；
2. 只有少数真正与时间压缩增强互补的信息才值得保留；
3. `SE1` 是当前保留下来的那一个；
4. 旧的 `133` 维与树特征选择线路仍有负证据价值，它们共同说明：

## 14. 检索路线：从 raw-label KNN 到 Retrieval V2.1

### 14.1 第一版 raw-label KNN

最初的想法是：

1. 从 `Timer-XL` 倒数第二层提取 `shared_repr`
2. 在训练库里按相似度找近邻
3. 直接用近邻真实磨损值做投票

这条线失败了。

失败原因包括：

1. `KNN` 本质是插值，不会外推；
2. 直接投票绝对磨损值容易被训练库上界钉死；
3. raw label vote 会同时拖坏 `mid/late`

### 14.2 Reality Check

后来项目明确承认了一个关键物理现实：

1. `KNN-only` 不会解决晚期超出训练上界的外推问题；
2. 它最现实的价值不是“预测新极值”，而是：
   - 给晚期提供经验上界兜底；
   - 抑制回归头在未见工况上的崩塌

### 14.3 检索目标重构

项目随后把检索对象从：

- `raw wear`

改成：

- `future wear delta`

即：

1. 不再检索未来绝对值；
2. 改为检索“相似历史片段的未来增量”

这一步是 `Retrieval V2.1` 成立的前提。

### 14.4 第一版 delta retrieval

第一版融合公式是：

- `pred_final = pred_head + beta * delta_knn`

这版仍然不够好，后来被认定为：

- “把整段增量又叠加了一遍”

### 14.5 Retrieval V2.1

最终成立的版本是：

1. 先用 `Current-best` 得到基础预测 `pred_head`
2. 计算：
   - `delta_head = pred_head - current_last_wear`
3. 用 `shared_repr` 在训练域晚期子库里做相似片段检索
4. 得到：
   - `delta_knn`
5. 再做增量残差修正：
   - `pred_final = pred_head + beta * (delta_knn - delta_head)`

这一步的含义是：

- **检索不是替代回归头，而是专门修正回归头对未来增量的偏差。**

## 25. 一页式结论

可以把整个项目压缩成下面这段话：

本项目围绕 `PHM2010` 刀具磨损跨工况预测展开，初始基线在 `c1,c4 -> c6` hardest fold 上表现最差。通过诊断发现，`C6` 与 `C1/C4` 在磨损演化上存在明显时间尺度错位与晚期分布失配。基于此，项目首先提出 `A2-Random-S2`，即仅使用 `stride=2` 的训练时时间压缩增强，以缓解时间节奏失配；随后引入 `SE1` 作为与时间增强正交的谱结构信息，形成 `Current-best = RMS7 + Feat_4 + A2-Random-S2 + SPECTRAL_ENTROPY_CH1`。进一步分段分析表明，剩余误差主要集中在 hardest fold 的 `late-stage` 系统性低估。围绕这一问题，`CORAL / MMD / AsymLoss` 等损失侧改造未能形成稳定收益。最终，项目转向测试时检索增强，提出 `Retrieval V2.1`：基于 `Timer-XL` 倒数第二层共享表示 `shared_repr` 做晚期子库检索，不直接投票未来绝对磨损，而是检索相似历史片段的未来增量，并通过 `delta residual blend` 修正回归头的增量偏差。正式配置为 `q80 + K=5 + cosine + beta=0.5`，三折平均达到 `MAE=3.7452, RMSE=5.0646`，显著优于 `Current-best`。当前主线已经收敛，但结果属于 `history-aware wear forecasting`，不是严格 `sensor-only online prediction`；后续论文阶段最需要补的是任务边界、文献协议对齐、`C1/C4/C6` 差异证据、导师建议回扣和投稿方向整理。

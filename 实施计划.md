# 实施计划.md（重建版）

更新时间：2026-03-16

## 一、先纠正共识（非常重要）
1. `c1,c4 -> c6` 这种“训练在 c1/c4，测试在 c6”的设置，本身**不构成数据泄露**。
2. 是否泄露的判断标准不是“有没有用到 c6”，而是：
   - 当前测试域是否参与了任何拟合或选择决策。
3. 因此正确约束应为：**当前折的测试域封存**，不是“c6 永久封存”。

## 二、实验总方案：三折跨工况（LOCO）
固定三组实验，三组都要做：
1. Fold-1：训练 `c1,c4`，测试 `c6`
2. Fold-2：训练 `c1,c6`，测试 `c4`
3. Fold-3：训练 `c4,c6`，测试 `c1`

说明：每一折里，被测试的那一把刀是“当前测试域”，必须封存。

## 三、零泄露硬约束（按“当前折”生效）
对每一折，`test_run` 一律禁止参与以下过程：
1. 特征提取规则拟合（若提取过程含可学习参数）。
2. 特征选择与打分（树重要性、相关性排序、RFE 等）。
3. 标准化/归一化/缺失值填充参数拟合。
4. 超参数搜索、早停、模型选择、融合权重确定。
5. 任何使用标签的统计或监督信号构建。

只允许两件事：
1. 模型冻结后，对 `test_run` 做一次最终盲测。
2. 盲测后做结果可视化与误差分析（不允许回流改模型）。

## 四、每一折的统一执行流程
1. 数据划分：
   - 训练域（两把刀）内部按 `8:2` 切分 train/val。
   - `test_run` 全程不参与切分和调参。
2. 特征工程（仅训练域）：
   - 原始数据 -> 候选特征提取。
   - 稳定性过滤（只在训练域内部比较）。
   - 冗余过滤（高相关去重）。
   - 预测性排序（树/单变量回归/RFE，均只用训练域）。
3. 训练：
   - 保持现有 TimerXL 算法与损失形式不改。
   - 双 DataLoader 仅在训练域两把刀之间混合。
4. 选择模型：
   - 仅根据 val 指标确定模型与超参。
5. 最终评估：
   - 在 `test_run` 上一次性输出 window/fullcurve/fullcurve_raw 指标。

## 五、特征选择策略（从“普通 top-k”升级）
不再直接“全量候选 -> tree top-k”。改为两阶段：

### 阶段 A：稳定性过滤（训练域内部）
1. 删除跨训练域偏移过大的特征（按标准化均值偏移阈值）。
2. 删除与 wear 关系方向不一致、波动过大的特征。
3. 删除高冗余特征（例如相关系数 > 0.98 只留一个）。

### 阶段 B：预测性选择（在 A 的剩余特征上）
1. 按树重要性/单变量回归/RFE 排序。
2. 首轮优先试 `k=6/8/10`，不再固定死 `k=16`。
3. 以验证集和跨折稳定性共同决定最终特征集。

## 六、训练与评估协议（统一口径）
1. 模型：保持当前 TimerXL 结构与物理约束损失，不改算法。
2. 数据加载：双 DataLoader，训练域两把刀分别构窗后混合采样。
3. 指标：
   - 主报告：`window/fullcurve/fullcurve_raw` 的 MSE/MAE。
   - 补充：跨折均值与标准差。
4. 可视化：每折输出窗口预测图、全曲线图、测试结果目录。

## 七、验收标准
1. 三折全部完成，且每折都有独立日志与图像产物。
2. 每折都能提供“零泄露检查记录”（见下一节）。
3. 汇总表包含：
   - 每折指标
   - 三折均值±方差
   - 特征数量与特征名单
4. 可明确回答：
   - 哪组特征在跨折最稳定。
   - 是否优于“原始 tree-k16”基线。

## 八、每折必填的零泄露检查清单
每次开跑前/后都记录以下 5 项：
1. 当前 `test_run` 是否参与任何拟合步骤：必须为“否”。
2. 特征选择是否仅在训练域完成：必须为“是”。
3. 标准化参数是否仅由训练域拟合：必须为“是”。
4. 模型选择/早停是否未参考 `test_run`：必须为“是”。
5. `test_run` 是否仅用于最终一次盲测：必须为“是”。

## 九、立即执行顺序（当前回合）
1. 先做 Fold-1：`c1,c4 -> c6`（按新策略重跑，严格零泄露）。
2. 你验收 Fold-1 后，再做 Fold-2：`c1,c6 -> c4`。
3. 最后做 Fold-3：`c4,c6 -> c1`。

## 1. 结论先行

当前结果可以分成两层：

- 正式主线结果：以 `paper_exec/csv` 与 `final_main_assets/tables` 为准，已经满足 clean protocol 与 inner-LOSO train-only 选参要求，可直接用于论文正文。

如果只保留一句话总结当前正式主线，则应写为：

在 PHM2010 三折 LOCO 跨工况评估中，`TimerXL head-only -> TimerXL + TMA -> TimerXL + TMA + KNN-DRR` 形成稳定提升链，平均 MAE 从 `8.7711` 降至 `6.1484`，再降至 `3.1784`；代表性困难折的 late-stage MAE 从 `21.2180` 降至 `5.7781`。

## 2. 正式主线结果（论文应以此为准）

### 2.1 三折主结果

来源：

- [main_results_3fold.csv](/home/jyc23/zml25/Timer/FeatureExact3.10/FeatureTest/OpenLTM/paper_exec/csv/main_results_3fold.csv)
- [表2_主结果.md](/home/jyc23/zml25/Timer/FeatureExact3.10/FeatureTest/OpenLTM/paper_exec/final_main_assets/tables/表2_主结果.md)

三折平均结果：

| 方法 | MAE | RMSE |
| --- | ---: | ---: |
| TimerXL head-only | 8.7711 | 11.9747 |
| TimerXL + TMA | 6.1484 | 9.4630 |
| TimerXL + TMA + KNN-DRR | 3.1784 | 3.9763 |

按折结果：

| 方法 | fold1 | fold2 | fold3 |
| --- | ---: | ---: | ---: |
| TimerXL head-only | 13.3984 | 6.9747 | 5.9401 |
| TimerXL + TMA | 9.2761 | 3.7426 | 5.4264 |
| TimerXL + TMA + KNN-DRR | 3.2065 | 3.3330 | 2.9958 |

解读：

- `TMA` 在三折平均层面带来第一段收益，说明训练期时间尺度增强确实改善了跨工况表示可比性。
- `KNN-DRR` 带来第二段更大的收益，是当前正式主线中的主要性能提升来源。
- 该提升链在三折上都成立，说明不是单折偶然现象。

### 2.2 代表性困难场景

来源：

- [表3_最困难场景.md](/home/jyc23/zml25/Timer/FeatureExact3.10/FeatureTest/OpenLTM/paper_exec/final_main_assets/tables/表3_最困难场景.md)

代表性困难折 `c1,c4 -> c6` 的关键结果：

| 模式 | MAE | RMSE |
| --- | ---: | ---: |
| TimerXL head-only | 13.3984 | 20.4362 |
| TimerXL + TMA | 9.2761 | 17.5968 |
| retrieval head-only | 9.2456 | 15.0871 |
| knn-only@k10 | 25.8334 | 40.5427 |
| delta-knn-only@k10 | 2.5154 | 3.2810 |
| delta-blend@k10_b07 | 3.2065 | 4.1586 |

解读：

- 纯 `knn-only@k10` 明显失败，说明脱离 backbone 趋势直接做检索并不稳健。
- `delta-knn-only@k10` 在该折上数值最好，但正文主线仍保留 `delta-blend`，因为 `delta residual blend` 的方法叙事更稳，更符合 formal 主表定位。
- 困难折结果说明，当前问题的关键不在于整条曲线的均匀偏移，而在于后期增量估计偏差。

### 2.3 分阶段残差证据

来源：

- [表4_阶段残差对比.md](/home/jyc23/zml25/Timer/FeatureExact3.10/FeatureTest/OpenLTM/paper_exec/final_main_assets/tables/表4_阶段残差对比.md)

fold1 的最关键 late-stage 结果：

| 模式 | late MAE | late mean residual | underest ratio |
| --- | ---: | ---: | ---: |
| head-only | 21.2180 | -20.4768 | 0.8493 |
| delta-knn-only@k10 | 4.0187 | 0.5214 | 0.6575 |
| delta-blend@k10_b07 | 5.7781 | -5.7781 | 1.0000 |
| baseline_head-only | 32.4302 | -32.4302 | 1.0000 |
| tma_head-only | 25.6950 | -24.9983 | 0.8767 |

解读：

- `baseline_head-only` 的 late-stage MAE 高达 `32.4302`，说明未引入 clean formal 主线前，后期低估更严重。
- `TimerXL head-only` 仍存在明显 late-stage 负残差（`-20.4768`），说明 backbone 本身尚不能消除结构性偏差。
- `TMA` 能缓解一部分问题，但不能单独解决 late-stage 低估。
- `delta-blend` 与 `delta-knn-only` 都能大幅减轻该结构性误差，这正是 KNN-DRR 被定位为主创新模块的核心证据。

### 2.4 Retrieval 消融

来源：

- [retrieval_ablation.csv](/home/jyc23/zml25/Timer/FeatureExact3.10/FeatureTest/OpenLTM/paper_exec/csv/retrieval_ablation.csv)

按折结果：

- `delta-blend@k10_b07` 虽不总是单折最优，但作为正式主线更稳妥，因为它保留了 backbone 趋势并执行受控残差修正。

## 3. 历史/探索性结果（来自结果.md）

### 3.1 full133 长训结果

`full133` 在 Fold1 上的 3-seed 平均：

| 对比组 | window MAE | fullcurve MAE | fullcurve_raw MAE |
| --- | ---: | ---: | ---: |
| dual 平均 | 38.4646 | 26.0453 | 57.4520 |
| single 平均 | 38.6957 | 26.1470 | 57.5600 |

结论：

- `dual` 与 `single` 性能很接近，`dual` 略优。
- 这一组结果可作为旧特征路线中的稳定基线，但明显不如当前 formal 主线。

### 3.2 td28 tree top16

3-seed 平均：

| 对比组 | window MAE | fullcurve MAE | fullcurve_raw MAE |
| --- | ---: | ---: | ---: |
| td28tree_k16_dual | 68.9865 | 50.5504 | 27.6291 |
| full133_dual | 38.4646 | 26.0453 | 57.4520 |

相对 `full133_dual`：

- `window MAE`：`+79.35%`
- `fullcurve MAE`：`+94.09%`
- `fullcurve_raw MAE`：`-51.91%`

结论：

- `td28tree_k16` 在 `fullcurve_raw` 上非常强，但在主口径 `window/fullcurve` 上显著落后。
- 这说明它不是一个适合当前论文主线的统一优解。

### 3.3 stageB k6/k8/k10

`seed2026` 下：

| setting | window MAE | fullcurve MAE | fullcurve_raw MAE |
| --- | ---: | ---: | ---: |
| k=6 | 19.1933 | 20.0115 | 20.0115 |
| k=8 | 20.1886 | 21.1193 | 21.1193 |
| k=10 | 21.2097 | 21.9902 | 21.9902 |

结论：

- 三种口径都是 `k=6` 最优。
- 随着 `k` 增大，指标单调变差。
- 在旧特征路线里，`stageB k=6` 是很强的一组结果。

### 3.4 采样均衡修复

`td28stageB_k6_dual_seed2026` 对比均衡采样版：

| setting | window MAE | fullcurve MAE | fullcurve_raw MAE |
| --- | ---: | ---: | ---: |
| 旧采样 | 19.1933 | 20.0115 | 20.0115 |
| 均衡采样 | 19.3694 | 20.2406 | 20.2406 |

结论：

- 修复了采样结构问题，但性能略降。
- 因此该修复更像工程一致性修补，而不是性能提升来源。

### 3.5 RMS7 + range7

| setting | window MAE | fullcurve MAE | fullcurve_raw MAE |
| --- | ---: | ---: | ---: |
| RMS7 + range7 | 22.5311 | 23.1108 | 23.1108 |
| td28stageB_k6_dual | 19.1933 | 20.0115 | 20.0115 |

结论：

- `RMS7 + range7` 不如 `stageB k6`。
- 它可作为补测记录保留，但不应抬升为当前主线。

### 3.6 batch size 对照

`td28tree_k16_dual_seed2026`：

| 指标口径 | bt96 MAE | bt32 MAE | 变化 |
| --- | ---: | ---: | ---: |
| window | 68.5977 | 70.1414 | +2.25% |
| fullcurve | 50.6663 | 53.1519 | +4.91% |
| fullcurve_raw | 27.5953 | 30.0787 | +9.00% |

结论：

- 在当前协议下，`batch_size 96 -> 32` 没有提升性能，反而全面变差。

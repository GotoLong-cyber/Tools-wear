# 实施计划2.md（重建版）

更新时间：2026-03-17

## 一、先纠正共识（非常重要）
1. `c1,c4 -> c6` 这种“训练在 c1/c4，测试在 c6”的设置，本身**不构成数据泄露**。
2. 是否泄露的判断标准不是“有没有用到 c6”，而是：
   - 当前测试域是否参与了任何拟合或选择决策。
3. 因此正确约束应为：**当前折的测试域封存**，不是“c6 永久封存”。

## 二、实验总方案：三折跨工况（LOCO）
固定三组实验，三组都要做：
1. Fold-1：训练 `c1,c4`，测试 `c6`
2. Fold-2：训练 `c1,c6`，测试 `c4`
3. Fold-3：训练 `c4,c6`，测试 `c1`

说明：每一折里，被测试的那一把刀是“当前测试域”，必须封存。

## 三、零泄露硬约束（按“当前折”生效）
对每一折，`test_run` 一律禁止参与以下过程：
1. 特征提取规则拟合（若提取过程含可学习参数）。
2. 特征选择与打分（树重要性、相关性排序、RFE 等）。
3. 标准化/归一化/缺失值填充参数拟合。
4. 超参数搜索、早停、模型选择、融合权重确定。
5. 任何使用标签的统计或监督信号构建。
6. 任何使用 `test_run` 无标签分布来决定阈值、排序、筛选规则或特征去留。

只允许两件事：
1. 模型冻结后，对 `test_run` 做一次最终盲测。
2. 盲测后做结果可视化与误差分析（不允许回流改模型）。

## 四、每一折的统一执行流程
1. 数据划分：
   - 训练域（两把刀）内部按 `8:2` 切分 train/val。
   - `test_run` 全程不参与切分和调参。
2. 特征工程（仅训练域）：
   - 原始数据 -> 候选特征提取。
   - 稳定性过滤（只在训练域及跨折训练域比较）。
   - 冗余过滤（高相关去重）。
   - 预测性排序（树/单变量回归/RFE，均只用训练域）。
3. 训练：
   - 保持现有 TimerXL 算法与损失形式不改。
   - 双 DataLoader 仅在训练域两把刀之间混合。
4. 选择模型：
   - 仅根据 val 指标确定模型与超参。
5. 最终评估：
   - 在 `test_run` 上一次性输出 window/fullcurve/fullcurve_raw 指标。

## 五、特征选择策略（从“普通 top-k”升级）
不再直接“全量候选 -> tree top-k”。改为两阶段：

### 阶段 A：稳定性过滤（训练域内部拟合 + 跨折训练域一致性）
1. 在当前折训练域内执行稳定性过滤，并汇总跨折训练域的一致性统计（不使用当前折 `test_run`）。
2. 删除跨训练域偏移过大的特征（按标准化均值偏移阈值）。
3. 删除与 wear 关系方向不一致、波动过大的特征（先在训练域内判定，再做跨折一致性复核）。
4. 删除高冗余特征（例如相关系数 > 0.98 只留一个）。
5. 产出两份结果：
   - 严格零泄露可训练特征集（仅由训练域决定）；
   - 跨折训练域一致性分析报告（用于解释与后续调参，不回流本折拟合）。

### 阶段 B：预测性选择（在 A 的剩余特征上，强调跨折/跨刀具稳定性）
1. 在每一折内先做预测性排序（树重要性/单变量回归/RFE），得到折内候选排名。
2. 不只看单折验证集表现，必须汇总跨折（`c1,c4->c6`、`c1,c6->c4`、`c4,c6->c1`）的一致性。
3. 特征入选标准改为“预测性 + 稳定性”双约束：
   - 折内有效（验证集表现不劣）；
   - 跨折/跨刀具排名稳定（波动小、方向一致）。
4. 首轮优先试 `k=6/8/10`，不再固定死 `k=16`。
5. 最终特征集由“跨折综合分”决定，而不是某一折最优分决定；当前折 `test_run` 不参与任何选择决策。

## 十四、下一步讨论版计划（2026-03-17，不执行）
当前共识：`RMS7 + Feat_4` 已验证“`+1` 增广”可行，下一步可以继续加特征，但必须从“盲目加维”改为“受控加维”。

### 14.1 目标
1. 目标不是把特征越堆越多，而是找到“对三折跨工况都稳”的最小增广集合。
2. 主比较对象固定为：
   - 基线A：`RMS7`
   - 基线B：`RMS7 + Feat_4`
3. 新方案必须在至少两折上优于基线B，且第三折不显著退化，才允许晋级。

### 14.2 候选特征来源（顶刊常见思路，先规划不执行）
在时域基础上，分三层候选池：
1. 形状类：`kurtosis/skewness/crest_factor/impulse_factor/shape_factor`。
2. 频域类：`band_energy_ratio/spectral_centroid/main_peak_freq`。
3. 时频类：`WPT节点能量占比/STFT高频能量占比`。

说明：
1. 候选池先建“公式与提取定义”，不做一次性大拼接。
2. 每个候选优先先做“单通道 +1”验证，再考虑多通道扩展。

### 14.3 评估与准入门槛（硬约束）
1. 严格零泄露：每折 `test_run` 不参与任何拟合、排序、阈值确定。
2. 双向训练域验证（折内）：
   - 方向A：`train=c1,val=c4`
   - 方向B：`train=c4,val=c1`
3. 准入门槛（单个候选）：
   - 两个方向 `MAE` 都不劣于基线B；
   - 且至少一个方向有显著下降（建议阈值 `>=2%`）。
4. 三折盲测门槛（进入结果汇总前）：
   - `hardest fold` 不劣于基线B；
   - 三折平均 `MAE` 改善；
   - 至少两折优于基线B；
   - 最差一折不超过基线B的 `+5%` 退化。

### 14.4 执行顺序（下一轮按此推进）
1. 第一步：定义候选特征公式表（只建表，不训练）。
2. 第二步：先做“单通道 +1”。
3. 第三步：通过门槛的候选，再试“多通道稀疏 +1”。
4. 第四步：若某一候选在多通道下仍稳定，再试“对称 +1”。
5. 第五步：通过门槛后再做 `+2` 组合测试（最多到 `+3`，避免维度膨胀）。
6. 第六步：按三折结果定稿“最终轻量特征集”。

### 14.5 当前结论（讨论版）
1. 不再默认“对称 +1 优先”，当前证据更支持“单通道受控增广优先”。
2. 不把 `e1000` 作为默认标准，后续筛选阶段必须统一轮次或统一早停策略。
3. 今后比较候选特征时，必须单独盯住 `hardest fold`；当前默认按 `c1,c4 -> c6` 作为 hardest fold 管理。
4. 可以继续加特征，但必须采用“逐个候选、逐级准入”的策略。
5. 当前优先级建议：先试形状类，再试频域，再试时频。

## 二十五、A2下一步消融计划（2026-03-24 00:10）
当前 `A2_dynstride` 已能说明“训练时动态步长增强有效”，但还不能充分说明“它为什么有效”。下一步不再直接扩特征，而是先补三组机制消融，回答收益来源问题。

### 25.1 消融目标
1. 证明当前收益不是“任何 stride 增强都有效”，而是“局部斜率引导规则有效”。
2. 证明当前收益主要来自 `stride=2` 还是 `stride=2+3` 的组合。
3. 区分当前收益到底来自：
   - 时间压缩（time compression）；
   - 还是高磨损阶段样本权重变化（stage/sample reweighting）。

### 25.2 消融A：随机 stride vs 斜率引导 stride
1. 目的：
   - 验证 `A2_dynstride` 的有效性是否真的来自“局部斜率规则”，而不是只要训练时加大步长就会提升。
2. 对照设置：
   - `A2-Slope`：当前版本，按局部斜率阈值决定是否允许 `stride>1`；
   - `A2-Random`：不按局部斜率，仅在训练窗口上随机分配 `stride=2/3`；
   - `Baseline`：固定 `stride=1`。
3. 口径要求：
   - 特征固定为 `RMS7 + Feat_4`；
   - 模型、损失、训练轮次、早停、GPU 设置全部不变；
   - 只改训练窗口生成策略；
   - `val/test` 必须继续固定 `stride=1`。
4. 随机对照的严格约束：
   - 不仅要保证“总增强样本数”与 `A2-Slope` 大致一致；
   - 还要保证各个步长的触发比例尽可能一致。
   - 例如：若 `A2-Slope` 在某折训练集中有 `30%` 窗口被赋予 `stride=2`、`1%` 窗口被赋予 `stride=3`，则 `A2-Random` 也必须按同等比例在全训练时间轴上随机分配。
   - 这样才能排除“只是增强数量更多”带来的干扰，纯粹比较“规则分布”本身是否有效。
5. 判据：
   - 若 `A2-Slope` 稳定优于 `A2-Random`，则说明“斜率规则”本身有贡献；
   - 若 `A2-Random` 和 `A2-Slope` 接近，则说明当前收益更像是“泛化增强数量效应”。

### 25.3 消融B：只用 `stride=2` vs `stride=2+3`
1. 目的：
   - 验证 `stride=3` 是否真的有必要保留。
2. 已知证据：
   - 当前诊断图显示，绝大多数增强窗口来自 `stride=2`；
   - `stride=3` 在当前阈值下触发极少。
3. 对照设置：
   - `A2-S2`：只允许 `stride=1,2`；
   - `A2-S23`：允许 `stride=1,2,3`（当前版本）。
4. 判据：
   - 若 `A2-S2` 与 `A2-S23` 持平或更优，则后续默认删掉 `stride=3`；
   - 若 `A2-S23` 在 hardest fold 上继续带来可重复收益，再保留 `stride=3`。

### 25.4 消融C：`dynstride` vs 阶段重采样
1. 目的：
   - 区分收益到底来自“时间压缩”还是“高磨损阶段样本权重变化”。
2. 对照设置：
   - `A2-Dynstride`：当前正式版本，训练时动态步长增强；
   - `A2-StageResample`：旧版训练域阶段重参数化方案，仅保留为诊断对照，不作正式主线；
   - `A2-StageWeightOnly`：不改时间轴，只提高高磨损阶段窗口的采样权重。
3. `A2-StageWeightOnly` 提升为必做项：
   - 原因：对导师和审稿人来说，“高磨损区重采样/重加权”是最自然的 baseline；
   - 若 `A2-Dynstride` 不能显著优于 `StageWeightOnly`，则当前“动态步长时间压缩”机制就还没有真正站稳。
4. 判据：
   - 若 `Dynstride` 明显优于 `StageWeightOnly`，说明收益不只是样本权重变化；
   - 若 `Dynstride` 与 `StageWeightOnly` 接近，则说明当前收益很可能主要来自高磨损区重加权，而不是时间尺度建模本身。

### 25.5 执行顺序
1. 第一优先：消融B（`stride=2` vs `2+3`）
   - 成本最低；
   - 可最快判断是否需要保留 `stride=3`。
2. 第二优先：消融A（`random` vs `slope-guided`）
   - 用来回答“规则是否有效”。
3. 第三优先：消融C（`dynstride` vs `stage resample/weight only`）
   - 用来回答“收益机制到底是什么”；
   - 这是论文写作和导师汇报最关键的一组证据。

### 27.5 当前收束结论
1. A2 正式口径继续保留 `S2`，不再保留 `stride=3`。

# Tools-wear
# **面向跨工况刀具磨损预测的时序大模型与检索增强方法研究**
3.25更新
## 1.数据分析

### 	1.1数据集

​		本研究采用的数据集为PHM2010，共有6个文件，分别为C1,C2,C3,C4,C5,C6，其中 C1,C4,C6带有真实的磨损标签。因此本次实验采用交叉验证的方法（c1,c4->c6;c1,c6->c4;c4,c6->c1，后续简称为fold1，fold2，fold3）。每个文件中包含315个子文件，每个子文件包含7个通道，分别为切削力（X/Y/Z），振动（X/Y/Z），声发射(AE)。真实磨损标签分为Flute1，Flute2，Flute3，本实验选择最大值作为磨损标签。

### 	1.2特征提取

​		主要提取的是数学统计量以及无量纲信息。实验初期主要是提取时域信息，分别在7个通道提取4个时域信息，分别提取了均方根、标准差、极差、偏度。实验中期提取采用了皮尔逊冗余去除的方法，分别提取了98和133个特征。实验后期提取了7个通道的RMS，以此作为实验基线，后续在基线的基础上增加极差以及频域特征的频谱熵。

​		实验初期，在提取7个时域特征时，发现模型无论是前段，中段，还是后端（后续简称为early；mid；late）都存在较大的误差，由于处于实验中期，大部分的实验都放在了特征提取上，经过28个特征不停的组合，模型依然很难收敛，很难到达理想的结果。但是通过提取7个通道的RMS，模型得到了较为显著的提升。在实验中期，考虑提取大量特征，在大量的特征中做皮尔逊冗余去除，去除掉高度相关特征，虽然模型在fold2和fold3上去得到比较明显的提升，但是在fold1中存在较为严重的域偏移。经分析，皮尔逊冗余去除只能更好的拟合训练工况，并且去除掉的特征有可能是作为fold1中强相关特征，只是在训练工况中没有表现出来。实验后期，我们采用基于树的特征提取，L1，以及KNN的方法来提取特征，在经过基于树的特征提取，基于之前提取的133个特征做全局TOP-K，每次选择K个特征进行验证，经过实验，最终选择了16个时域信息，在fold1上依然存在较为严重的域偏移。只要fold2和fold3的结果足够好，fold1的效果就越差。基于此现象，我们选择提取的7个通道的RMS作为我的基线，在此基础上增加特征，不再使用全局TOP-K的方式枚举，实验中每增加一个特征，fold1的效果就会变差。基于此，我们考虑分析数据的分布。

### 	1.3时间对齐

​		我们生成了C1,C4,C6的磨损曲线图。

![c1_c4_c6磨损曲线](D:\zmlProject\实验图片\c1_c4_c6磨损曲线.png)

​		在该图中，我们能够很明显的看到C6的磨损曲线是远远高于C1,C4的，同时C4存在一段较为缓慢的磨损。我们考虑是否是由于时间尺度没有对齐，导致模型在fold1中的测试中不能捕获磨损信息。如果还按照之前的方法喂给模型，不管提取多少特征，模型都不能拟合C6曲线，并且会产生严重的时间幻觉。于是我们提出了Random-S2 的时间对齐方法。简单来说，模型在学习数据的时候，由于fold1的磨损曲线之间存在较大的差异，此时模型可以通过看2倍数据的方式，来拟合曲线，在后续的实验中证明该方法有效。同时我们还验证了strid=3，即模型看3倍的数据，实验结果证明没有提升，即我们确定采用S2的方式来训练模型。随后我们想如果给late磨损较高的权重是否能解决当前问题，即当时间越晚，我们给高度磨损更高的权重，让模型更多的学习磨损趋势，但结果表明此方法不能解决跨域泛化的问题。因此时间对齐可以作为解决当前跨域问题的突破口。

### 	1.4时间对齐后增加特征

​		在时间对齐有明显的效果后，我们考虑继续按照之前的策略，增加特征，不过现在增加的频域特征，分别依次增加了频谱质心、高频率占比、频谱熵。经过实验我们认为频谱熵的效果在fold1、fold2、fold3中完全超过我们的基线。后续无论在增加多少 特征都不能超过频谱熵的效果。此时我们考虑通过手动特征提取方法已经快接近当前模型的天花板了。因此我们开始考虑做对齐的问题，我们共考虑三种方式，分别为CORAL、MMD、以及KNN。其中CORAL的结果较差，MMD的结果和Current-best基本持平，MMD无论如何微调在当前模型中都得不到提升。我们考虑KNN的离线检测，主要是基于fold1开展实验，使用C1,C4构造库，结果得到了显著的提升，当前已作为最终结果。

### 	1.5训练集和测试集构造

​			本实验采用三折交叉验证的方式，C1,C4,C6依次作为测试集，我们以fold1为例，当C1,C4作为训练集时，我们将C1,C4的数据进行分别构窗，然后再进行合并，合并后按照8:2的比例划分训练集和验证集，在训练过程中进行shuffle取样，C6作为测试集时，模型看不到任何C6的信息。

## 2.模型

​	本次研究采用的Timer-xl模型，这里我们只做简单讲解，后续会补充。

​	Timer-xl作为当前较为主流的时序数据大模型，并且在海量的时序数据上进行了预训练，本研究采用微调下游任务，来解决刀具磨损预测的问题。我们的实验协议主要是通过读取刀具前96个时间点的磨损值和传感器数值，来预测后16步。如果是作为严格的在线检测，那我们就不能这样考虑模型的输入，但是我们是**history-aware**，因此在这里我们不存在数据泄露。包括后面KNN建库，使用的第96个时间点的数据作为锚点出发。

输入输出协议（论文口径）：
1. 输入：过去 `96` 步的磨损值与传感器特征；
2. 输出：未来 `16` 步的磨损值；
3. 推理阶段仅访问历史观测窗口内的传感器信号，不使用预测时域内的未来传感器信息。

## 3.结果

1. 我们先确认了 fold1 的核心问题不是普通特征不足，而是 C6 存在明显时间尺度错位和晚期低估。
2. 然后用 A2-Random-S2 做训练时时间压缩增强，解决了时间节奏问题。
3. 再用 SE1 提供正交谱结构信息，形成 Current-best。
4. 在此基础上，CORAL/MMD/AsymLoss 都没有形成稳定收益。
5. 最后我们在测试时基于 Timer-XL 倒数第二层共享表示做检索，用 delta residual blend 修正晚期增量偏差，形成 Retrieval V2.1。
6. 三折结果表明它对 hardest fold 提升最显著，且三折平均最优。
7. 但当前协议属于 history-aware forecasting，不是严格 sensor-only online prediction。

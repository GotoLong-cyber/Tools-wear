- `delta-blend MAE` 从 `2.9350` 降到 `2.5104`，说明在更干净的 `no-val` 训练协议下，固定 retrieval 配置仍然有效，而且整体更优。
- 唯一变差的是 `delta-knn-only`：其三折平均从 `2.2514` 升到 `2.4428`。这说明无验证集训练虽然提升了直接预测质量，但并不必然让检索库的几何结构同步变优。
- 因此，这组补充实验支持将 `no validation set + fixed epochs` 视为当前数据规模下更合理的 backbone 训练协议；若后续正式切换论文主协议，应优先报告 `head-only` 与 `delta-blend` 两条结果链，并把 `delta-knn-only` 的轻微回退作为补充讨论。

设置：

- 仅在最困难折 `fold1: C1+C4 -> C6` 上做代表性消融。
- 协议固定为 `wear_agg=mean`、`no validation set`、`split_ratio=1.0`、`time_gap=0`。
- retrieval 配置固定为 inner-LOSO train-only 选出的 `k=10, beta=0.7, late_q=0.0`。
- 目的不是寻找单点最优，而是检验无验证集盲训策略对 epoch 选择是否敏感。

结果：

| Epoch | head-only MAE | delta-knn-only MAE | delta-blend MAE |
| ---: | ---: | ---: | ---: |
| 50 | 3.6613 | 2.3883 | 2.1006 |
| 100 | 3.2097 | 2.3961 | 2.2673 |
| 150 | 3.0119 | 2.4042 | 2.3723 |
| 200 | 2.8229 | 2.4102 | 2.3019 |
| 250 | 2.7047 | 2.4125 | 2.2643 |
| 300 | 2.9885 | 2.4251 | 2.2784 |

结论：

- `head-only` 随训练轮数增加呈现“先改善、后趋稳”的趋势，在当前折上约于 `epoch=250` 达到最好值 `2.7047`，之后到 `300` 轮仅轻微回升。
- `delta-knn-only` 几乎不受 epoch 影响，6 个节点都稳定在 `2.39~2.43` 的窄区间内，说明 retrieval 本身对 backbone 训练轮数具有较强脱敏性。
- `delta-blend` 同样表现出明显的 epoch 不敏感特征，6 个节点都维持在 `2.10~2.37` 区间，没有出现因缺少 validation 而导致的剧烈退化。
- 因此，fold1 消融支持“固定 epoch 的无验证集盲训”具有较强稳定性这一主张。就当前节点看，`epoch=200` 并非唯一最优点，但处于一个足够稳健的平坦区间内，适合作为统一训练轮数使用。

## 6. 最终总判断

如果把“所有实验”放在一起看，结论是：

1. 当前**最强且最干净**的主线，是 `TimerXL backbone + TMA + KNN-DRR` 的三折 formal 结果。
2. 当前**最关键的协议实验**，是 inner LOSO train-only 选参；它决定了 `k=10, beta=0.7, late_q=0.0` 这一固定配置。
3. 当前**最重要的机制证据**，是 hardest fold 与 stage-wise residual，尤其是 late-stage underestimation 的大幅缓解。
4. 当前**仍不完整的增强资产**，是 `full133_vs_selected` 与 `backbone_comparison`，它们可以保留，但不能抬升为主结论。
5. 当前**历史特征路线**中最强的一支是 `stageB k=6`，但它已经不再是投稿主线。

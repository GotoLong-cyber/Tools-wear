# 论文风险与待补闭环清单

## 1. `A2` 的准确含义
1. `A2` 不是公开通用缩写，而是本项目内部的方法版本编号。
2. 在当前定版语境下：
   - `A2 = S2-only train-time time compression augmentation`
   - 即：**仅使用 `stride=2` 的训练时时间压缩增强**。
3. 当前正式实现采用：
   - `A2-Random-S2`
4. 其机制结论已经固定为：
   - 有效因素是 `stride=2` 的时间压缩增强本身；
   - `stride=3` 无贡献；
   - `slope-guided` 规则未证明优于随机分配；
   - 单纯高磨损样本重加权不能复现收益。

## 2. 当前最该补的 5 个漏点

### 漏点1：任务边界仍有张力
1. 导师要求：测试时不能看到测试机的任何东西。
2. 当前正式最优 `Retrieval V2.1` 实际协议是：
   - 使用测试窗口历史输入；
   - 使用测试窗口最后一个已观测 wear；
   - 不使用测试未来真实标签参与建库、检索或融合。
3. 这意味着当前结果属于：
   - `history-aware wear forecasting`
4. 但它**不是**严格：
   - `sensor-only online prediction`
5. 这是目前最大的方法学边界风险，答辩和论文里必须单独写清。

### 漏点2：文献对比还缺“协议对齐表”
1. 现在已经有文献结果表，但还不够。
2. 还需要一张专门的协议对齐表，至少包含：
   - 数据集是否完全一致；
   - 是否 `leave-one-tool-out`；
   - 是否使用历史 wear 作为输入；
   - 预测目标是单点还是整曲线；
   - 指标是 window 还是 fullcurve/raw；
   - 是否能做硬对比。
3. 否则“优于现有方法”这句话还不够严谨。

### 漏点3：`C1/C4/C6` 差异分析还没整理成论文级证据
1. 导师最关心的是：
   - `C6` 和 `C1/C4` 到底差在哪。
2. 现在虽然已有：
   - 分段误差表；
   - `A2` 消融；
   - `SE1` 正交信息；
   - `Retrieval V2.1` 晚期修复。
3. 但还缺一套单独的差异证据图表：
   - wear 曲线归一化对齐；
   - 各刀具 late-stage 斜率/增量分布；
   - `shared_repr` 跨刀具距离或聚类分布；
   - `C6` 晚期窗口与训练库最近邻的距离分布。
4. 这套证据会直接支撑“时间尺度错位 + 晚期分布失配”的主叙事。

### 漏点4：导师提出的“后段截取/检索思路”还没明确回扣
1. 导师提出过非常接近当前主线的想法：
   - 截取更接近 `C6` 晚期的训练片段；
   - 用检索增强来补 hardest fold 晚期样本覆盖。
2. 现在实验上其实已经通过 `Retrieval V2.1` 实现了这一点。
3. 但文档叙事上还没有明确写成：
   - `Retrieval V2.1` 是对导师“late-part retrieval/augmentation”思路的正式验证。
4. 这个回扣要补上，否则实验做到了，论文故事却断了一截。

### 漏点5：投稿方向与期刊层级还没系统整理
1. 导师明确要求：
   - 看别人投什么期刊；
   - 看当前工作适合投什么层级。
2. 现在文献结果已整理，但还没有形成：
   - 目标期刊清单；
   - 每个期刊的主题匹配度；
   - 当前工作与期刊风格的差距项。
3. 如果准备进入论文阶段，这一块必须尽快补上。

## 3. 当前优先级建议
1. 第一优先：单独写清当前协议边界（history-aware vs sensor-only）。
2. 第二优先：补文献协议对齐表。
3. 第三优先：补 `C1/C4/C6` 差异证据图。
4. 第四优先：把 `Retrieval V2.1` 回扣成导师建议的 late-part retrieval 验证。
5. 第五优先：整理投稿目标与期刊层级。

## 4. 一句话结论
当前主线结果已经够强，下一阶段最缺的不是新实验，而是把：
- 任务边界
- 文献对齐
- 差异证据
- 导师建议回扣
- 投稿方向
这 5 个闭环补齐。

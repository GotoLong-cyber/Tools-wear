# OpenLTM 环境说明

本文档只说明当前 `Timer/FeatureExact3.10/FeatureTest/OpenLTM` 主线实验所依赖的运行环境，不包含数据集与实验结果文件。

## 1. 适用范围

本说明适用于当前这条正式主线：

- `RMS7 + Feat_4 + A2-Random-S2 + SPECTRAL_ENTROPY_CH1 + Retrieval V2.1`

对应仓库根目录：

- `/home/lc24/Timer/FeatureExact3.10/FeatureTest/OpenLTM`

## 2. 当前实际使用的 Python 环境

项目脚本当前默认绑定的解释器是：

- `/home/lc24/miniconda3/envs/TimerXL/bin/python`

本机当前存在的 conda 环境包括：

- `base`
- `NewEnvName`
- `Timer`
- `TimerXL`
- `llm-hownet`
- `llm-ner`
- `psloss`

其中，`OpenLTM/TimerXL` 主线实验实际使用的是：

- `TimerXL`

## 3. 当前已核到的核心版本

在 `TimerXL` 环境中，当前实际版本为：

- `Python 3.11.14`
- `PyTorch 2.9.1+cu128`
- `torch.version.cuda = 12.8`
- `numpy 2.3.5`
- `pandas 2.2.3`

这意味着：

- 当前训练环境绑定的是 **CUDA 12.8**
- 不是 `11.x`

## 4. 对 GPU / CUDA 的要求

### 4.1 当前环境的硬要求

由于当前 PyTorch 是：

- `torch 2.9.1+cu128`

所以目标服务器至少需要：

- NVIDIA 驱动能够兼容 `CUDA 12.8`

更直白地说：

- `CUDA 12.8 / 12.9 / 更高兼容驱动` 可以直接承接当前环境
- 如果新服务器只有 `11.x` 驱动能力，则当前这套环境 **不能直接用**

### 4.2 如果新服务器只有 11.x

那就不能原样搬迁当前 `TimerXL` 环境。

需要做的是：

1. 重新创建新的 conda 环境
2. 重装与 `11.x` 兼容的 PyTorch 版本，例如 `cu118`
3. 再做一个最小回归实验，确认新旧环境结果差异

## 5. 当前脚本对环境路径的默认假设

项目里大量脚本直接写死了解释器路径，例如：

- `PYTHON_BIN="/home/lc24/miniconda3/envs/TimerXL/bin/python"`

因此如果迁移到新机器，至少有两种办法：

### 方法 A：保留同样的环境名和目录结构

即仍然创建：

- `/home/lc24/miniconda3/envs/TimerXL`

优点：

- 现有脚本基本不用改

### 方法 B：换新环境路径，但统一修改脚本

需要批量修改所有相关脚本中的：

- `PYTHON_BIN`

优点：

- 环境命名更自由

缺点：

- 容易漏改

当前更推荐：

- **方法 A**

## 6. 当前仓库不能保证“下载即跑”的原因

当前仓库已经能保证：

- 代码不丢
- 脚本不丢
- 文档不丢

但还不能保证任意机器 `git clone` 后直接运行，原因主要有：

1. 依赖环境没有完全锁死为可直接恢复的环境文件
2. 脚本依赖本地解释器绝对路径
3. 数据集和实验结果目录没有随仓库提交
4. 目标机器的 CUDA / 驱动能力必须兼容 `cu128`

## 7. 当前仓库之外仍需准备的内容

如果要在新机器继续运行 `OpenLTM`，至少还需要自行准备：

### 7.1 数据

- `dataset/`

### 7.2 如需继续复现实验或做检索增强评估

还需要已有训练产物，例如：

- `checkpoint/`
- `checkpoints/`
- 已训练模型目录

### 7.3 如需继续看正式结果

仓库未包含：

- `results/`

因此如果要在新机器上直接查看或复用旧实验结果，需要单独迁移这些目录。

## 8. 迁移到新服务器时的最小建议流程

建议按以下顺序做，不要一上来就重跑全部三折。

1. 创建新环境
2. 安装与目标服务器驱动兼容的 PyTorch
3. 保持环境名为 `TimerXL`
4. 把项目代码同步到新机器
5. 准备好 `dataset/` 和必要 `checkpoint/`
6. 先做一个最小回归实验

推荐的最小回归实验可以是：

- `fold1 current-best`
或
- `fold1 Retrieval V2.1`

先比较：

- `MAE`
- `RMSE`
- 曲线形状

如果差异可接受，再继续迁移全部实验。

## 9. 当前最重要的环境结论

一句话概括：

- 当前 `OpenLTM` 主线运行环境是 `TimerXL` conda 环境
- 核心版本是 `PyTorch 2.9.1+cu128`
- 目标服务器需要支持 **CUDA 12.8 及以上兼容驱动**
- 如果只有 `11.x`，则必须重建环境，不能直接原样运行
